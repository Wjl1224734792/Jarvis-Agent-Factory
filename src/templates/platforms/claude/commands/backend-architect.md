# 后端架构师对话模式

立即执行：

1. 加载基座技能：
   - `Skill("behavioral-guidelines")`

2. 代码注释语言：遵从 `behavioral-guidelines` 准则 5（注释语言约定）。

3. 了解用户当前面临的后端架构问题：
   - 项目背景（新项目启动 / 单体拆分 / 性能优化 / 可靠性改造）
   - 当前技术栈和团队能力
   - 核心痛点（可用性、一致性、扩展性、延迟...）
   - 用户是否已有倾向方案

4. 确认问题后，**必须调用 `Agent` 工具** spawn `backend-architect` 将完整上下文传递给它（不可绕过）：
   ```
   Agent(
     description="后端架构方案设计",
     subagent_type="backend-architect",
     prompt="<用户的问题描述、项目背景、技术栈约束、痛点，要求输出架构方案、ADR 和原型验证>"
   )
   ```

5. 将后端架构师的输出完整呈现给用户，必要时补充解释。

**关键纪律**（不可绕过）：
- 不要自己替代后端架构师做分析——必须通过 Agent 工具 spawn 它
- 不要在未确认问题边界的情况下直接 spawn
- 架构原型代码只做验证，不写入生产路径

---

## 红线
- 只输出方案不修改代码——架构咨询是只读模式
- 方案必须考虑分布式可靠性——单点故障不可接受
- 数据一致性方案必须明确——强一致性 vs 最终一致性必须有选择依据
- 架构方案必须含容量规划——预估 QPS、存储、带宽需求

## 目的

任何SaaS财务指标的快速参考，无需深入教学。当你需要快速公式查找、基准检查或决策框架提醒时使用。详细解释、计算和示例请参见相关的深入技能。

这不是教学工具——它是针对速度优化的速查表。扫描、查找、应用。

## 核心概念

### 指标类别

指标组织为四个族类：

1. **收入与增长** — 顶线金钱（收入、ARPU、ARPA、MRR/ARR、流失、NRR、扩展）
2. **单位经济** — 客户级别盈利能力（CAC、LTV、回收期、利润率）
3. **资本效率** — 现金管理（烧钱率、跑道、OpEx、净收入）
4. **效率比率** — 增长与盈利能力平衡（Rule of 40、magic number）

### 何时使用此技能

**在以下情况使用：**
- 你需要快速公式或基准
- 你正在准备董事会会议或投资者电话
- 你正在评估决策并需要检查哪些指标相关
- 你想快速识别红旗

**在以下情况不要使用：**
- 你需要详细的计算指导（使用 `saas-revenue-growth-metrics` 或 `saas-economics-efficiency-metrics`）
- 你第一次学习这些指标（从深入技能开始）
- 你需要示例和常见陷阱（相关技能中涵盖）

## 应用

### 所有指标参考表

| **指标** | **公式** | **它衡量什么** | **良好基准** | **红旗** |
|------------|-------------|----------------------|-------------------|--------------|
| **收入** | 扣除费用前的总销售额 | 赚取的顶线金钱 | 增长率 >20%同比（因阶段而异） | 收入增长慢于成本增长 |
| **ARPU** | 总收入 / 总用户数 | 每用户收入 | 因模式而异；跟踪趋势 | ARPU逐队列下降 |
| **ARPA** | MRR / 活跃账户 | 每客户账户收入 | SMB：$100-$1K；中型：$1K-$10K；企业：$10K+ | 高ARPA + 低ARPU（席位变现不足） |
| **ACV** | 每合同年化经常性收入 | 年化合同价值 | SMB：$5K-$25K；中型：$25K-$100K；企业：$100K+ | ACV下降（无意中下移市场） |
| **MRR/ARR** | MRR × 12 = ARR | 可预测的经常性收入 | 增长 + 质量重要；跟踪组成 | 新MRR下降而流失稳定/增长 |
| **流失率** | 流失客户 / 起始客户 | 取消的客户% | 月 <2%很好，<5%还行；年 <10%很好 | 流失逐队列增加 |
| **NRR** | (起始ARR + 扩展 - 流失 - 缩减) / 起始ARR × 100 | 收入留存 + 扩展 | >120%优秀；100-120%良好；90-100%还行 | NRR <100%（基础在缩减） |
| **扩展收入** | 升级 + 交叉销售 + 用量增长 | 来自现有客户的额外收入 | 总收入的20-30% | 扩展 <10%的MRR |
| **Quick Ratio** | (新MRR + 扩展MRR) / (流失MRR + 缩减) | 收入增益 vs. 损失 | >4优秀；2-4健康；<2漏水桶 | Quick Ratio <2（漏水桶） |
| **毛利率** | (收入 - COGS) / 收入 × 100 | 直接成本后收入的% | SaaS：70-85%良好；<60%令人担忧 | 毛利率 <60%或下降 |
| **CAC** | 总S&M支出 / 新客户 | 获取一个客户的成本 | 因企业而异：$10K+还行；SMB <$500 | CAC增加而LTV持平 |
| **LTV** | ARPU × 毛利率% / 流失率 | 来自一个客户的总收入 | 必须3x+ CAC；因细分而异 | LTV逐队列下降 |
| **LTV:CAC** | LTV / CAC | 单位经济效率 | 3:1健康；<1:1不可持续；>5:1投资不足 | LTV:CAC <1.5:1 |
| **回收期** | CAC / (月ARPU × 毛利率%) | 回收CAC的月数 | <12个月很好；12-18还行；>24令人担忧 | 回收期 >24个月（现金陷阱） |
| **贡献利润率** | (收入 - 所有可变成本) / 收入 × 100 | 可变成本后的真实贡献 | SaaS 60-80%良好；<40%令人担忧 | 贡献利润率 <40% |
| **烧钱率** | 月现金支出 - 收入 | 每月消耗的现金 | 早期净烧钱 <$200K可管理；增长 <$500K | 净烧钱加速 |
| **跑道** | 现金余额 / 月净烧钱 | 现金流耗尽前的月数 | 12+个月良好；6-12还行；<6危机 | 跑道 <6个月 |
| **OpEx** | S&M + R&D + G&A | 运营业务的成本 | 应比收入增长慢 | OpEx增长快于收入 |
| **净收入** | 收入 - 所有费用 | 实际利润/损失 | 早期负可以；成熟10-20%+利润率 | 损失加速而无增长 |
| **Rule of 40** | 收入增长% + 利润率% | 增长与效率的平衡 | >40健康；25-40还行；<25令人担忧 | Rule of 40 <25 |
| **Magic Number** | (Q收入 - 上Q收入) × 4 / 上Q S&M | S&M效率 | >0.75高效；0.5-0.75还行；<0.5修复GTM | Magic Number <0.5 |
| **经营杠杆** | 收入增长 vs. OpEx增长 | 规模化效率 | 收入增长 > OpEx增长 | OpEx增长快于收入 |
| **毛收入vs.净收入** | 净 = 毛 - 折扣 - 退款 - 积分 | 你实际保留的 | 退款 <10%；折扣 <20% | 退款 >10%（产品问题） |
| **收入集中度** | 前N客户 / 总收入 | 对最大客户的依赖 | 头号客户 <10%；前10 <40% | 头号客户 >25%（生存风险） |
| **收入构成** | 产品/细分收入 / 总收入 | 组合构成 | 无单一产品 >60%理想 | 单一产品 >80%（无多样化） |
| **队列分析** | 按加入日期分组客户；跟踪行为 | 业务是否在改善或恶化 | 近期队列与旧队列相同/更好 | 较新队列表现更差 |
| **按渠道CAC回收** | CAC / 月贡献（按渠道） | 按获取渠道的回收 | 跨渠道比较 | 一个渠道远比其他差 |
| **毛利率回收** | CAC / (月ARPU × 毛利率%) | 使用实际利润的回收 | 通常简单回收的1.5-2倍 | 使用利润率的回收 >36个月 |
| **单位经济** | 每单位收入 - 每单位成本 | 每个"单位"的盈利能力 | 需要正的贡献 | 负的贡献利润率 |
| **细分回收期** | CAC / 月贡献（按细分） | 按客户细分的回收 | 比较以分配资源 | 一个细分不盈利 |
| **增量性** | 由行动引起的收入 - 基线 | 营销/促销的真实影响 | 用保留组测试衡量 | 庆祝本就会发生的收入 |
| **营运资金** | 收入与收款之间的现金时间差 | 现金 vs. 收入时间差 | 年度预付 > 月度账单 | 长付款期限消耗跑道 |

### 快速决策框架

使用这些框架结合指标处理常见PM决策。

#### 框架1：我们应该构建这个功能吗？

**问：**
1. **收入影响？** 直接（定价、附加组件）或间接（留存、转化）？
2. **利润率影响？** COGS是什么？它会稀释利润率吗？
3. **ROI？** 收入影响 / 开发成本

**在以下情况构建：**
- 第一年ROI >3x（直接变现），或
- LTV影响 > 10x开发成本（留存），或
- 战略价值覆盖短期ROI

**在以下情况不要构建：**
- 即使乐观采用率下贡献利润率也为负
- 回收期超过平均客户生命周期

**需检查的指标：** 收入、毛利率、LTV、贡献利润率

---

#### 框架2：我们应该扩量这个获取渠道吗？

**问：**
1. **单位经济？** CAC、LTV、LTV:CAC比率
2. **现金效率？** 回收期
3. **客户质量？** 队列留存、按渠道的NRR
4. **可扩量性？** Magic Number、可触达的量

**在以下情况扩量：**
- LTV:CAC >3:1 且
- 回收期 <18个月 且
- 客户质量达到/超过其他渠道 且
- Magic Number >0.75

**在以下情况不要扩量：**
- LTV:CAC <1.5:1 且
- 无明确的改善路径

**需检查的指标：** CAC、LTV、LTV:CAC、回收期、NRR、Magic Number

---

#### 框架3：我们应该改变定价吗？

**问：**
1. **ARPU/ARPA影响？** 每客户收入会增加吗？
2. **转化影响？** 帮助或损害试用转付费转化？
3. **流失影响？** 创造流失风险还是降低它？
4. **NRR影响？** 启用扩展还是创造缩减？

**在以下情况实施：**
- 考虑流失风险后净收入影响为正
- 可以在广泛推出前对细分进行测试

**在以下情况不要改变：**
- 高流失风险而无抵消的扩展
- 在承诺前无法测试假设

**需检查的指标：** ARPU、ARPA、流失率、NRR、CAC回收期

---

#### 框架4：业务健康吗？

**按阶段检查：**

**早期阶段（$10M ARR以下）：**
- 增长率 >50%同比
- LTV:CAC >3:1
- 毛利率 >70%
- 跑道 >12个月

**增长阶段（$10M-$50M ARR）：**
- 增长率 >40%同比
- NRR >100%
- Rule of 40 >40
- Magic Number >0.75

**规模化阶段（$50M+ ARR）：**
- 增长率 >25%同比
- NRR >110%
- Rule of 40 >40
- 利润率 >10%

**需检查的指标：** 收入增长、NRR、LTV:CAC、Rule of 40、Magic Number、毛利率

### 按类别的红旗

#### 收入与增长红旗
| **红旗** | **这意味着什么** | **行动** |
|--------------|-------------------|------------|
| 流失逐队列增加 | 产品市场契合度在恶化 | 停止扩量获取；先修复留存 |
| NRR <100% | 基础在缩减 | 在扩量前修复扩展或减少流失 |
| 收入流失 > 客户流失 | 失去大客户 | 调查高价值客户为何离开 |
| Quick Ratio <2 | 漏水桶（勉强超过损失） | 在扩量获取前修复留存 |
| 扩展收入 <10%的MRR | 无升级/交叉销售引擎 | 构建扩展路径 |
| 收入集中度 >50%在前10客户 | 生存依赖风险 | 多样化客户基础 |

#### 单位经济红旗
| **红旗** | **这意味着什么** | **行动** |
|--------------|-------------------|------------|
| LTV:CAC <1.5:1 | 以亏损购买收入 | 在扩量前降低CAC或增加LTV |
| 回收期 >24个月 | 现金陷阱（长现金回收） | 协商年度预付或降低CAC |
| 毛利率 <60% | 每美元低盈利能力 | 提价或降低COGS |
| CAC增加而LTV持平 | 单位经济恶化 | 优化转化或减少销售周期 |
| 贡献利润率 <40% | 可变成本后不盈利 | 削减可变成本或提价 |

#### 资本效率红旗
| **红旗** | **这意味着什么** | **行动** |
|--------------|-------------------|------------|
| 跑道 <6个月 | 生存危机 | 立即融资或削减烧钱 |
| 净烧钱加速而无收入增长 | 更快烧钱而无结果 | 削减成本或增加收入紧迫性 |
| OpEx增长快于收入 | 负经营杠杆 | 冻结招聘；优化支出 |
| Rule of 40 <25 | 烧现金而无增长 | 改善增长或削减至盈利 |
| Magic Number <0.5 | S&M引擎损坏 | 在扩量支出前修复GTM效率 |

### 何时使用哪个指标

**功能优先级排序：**
- 收入影响 → 收入、ARPU、扩展收入
- 利润率影响 → 毛利率、贡献利润率
- ROI → LTV影响、开发成本

**渠道评估：**
- 获取成本 → CAC、按渠道CAC
- 客户价值 → LTV、按渠道NRR
- 回收 → 回收期、按渠道CAC回收
- 可扩量性 → Magic Number

**定价决策：**
- 变现 → ARPU、ARPA、ACV
- 影响 → 流失率、NRR、扩展收入
- 效率 → CAC回收期（定价变更会影响它吗？）

**业务健康：**
- 增长 → 收入增长、MRR/ARR增长
- 留存 → 流失率、NRR、Quick Ratio
- 经济 → LTV:CAC、回收期、毛利率
- 效率 → Rule of 40、Magic Number、经营杠杆
- 生存 → 烧钱率、跑道

**董事会/投资者报告：**
- 关键指标：ARR、收入增长%、NRR、LTV:CAC、Rule of 40、Magic Number、烧钱率、跑道
- 阶段特定：早期阶段强调增长+单位经济；增长阶段强调Rule of 40+Magic Number；规模化阶段强调盈利能力+效率

## 示例

### 示例1：功能投资合理性检查

你正在决定是否构建一个高级导出功能。

1. 使用框架1（我们应该构建这个功能吗？）
2. 拉取基线指标：ARPU、毛利率、LTV、贡献利润率
3. 建模乐观、基准和下行情境下的采用率
4. 如果在下跌情况下贡献利润率变为负，则拒绝

快速输出：
- 基准案例ROI：3.8x
- 贡献利润率影响：+4个百分点
- 决策：立即构建，推出后90天检查流失和扩展

### 示例2：渠道扩量决策

付费社交产生了很多注册但留存差。

1. 使用框架2（我们应该扩量这个获取渠道吗？）
2. 检查CAC、LTV:CAC、回收期和按渠道的NRR
3. 与最佳表现渠道比较，而非公司平均水平

快速输出：
- LTV:CAC：1.6:1
- 回收期：26个月
- NRR：88%
- 决策：不要扩量；封顶支出并运行定向优化测试

## 常见陷阱

- 使用混合公司平均而非队列或渠道级别指标
- 当Quick Ratio弱且留存恶化时扩量获取
- 将高LTV:CAC视为充分条件而不检查回收期和跑道影响
- 仅基于ARPU提升提价而不建模流失和缩减
- 跨不匹配的公司阶段或业务模式比较基准
- 跟踪许多指标而无明确的决策问题

需求与设计（5）

/ask
需求探询 — 4模式自适应：Interview(模糊→Socratic追问)、Direct(清晰→直接分析)、Consensus(ralplan式3阶段共识)、Review(Critic评估现有计划)。含5维度评分矩阵(BV/EF/RS/UI/DC)
K0→K1→K2→K3 allow_jump 评分矩阵
/research
深度研究 — RS0课题定义→RS1信息收集→RS2多维分析→RS3假设验证→RS4研究报告。竞争假设+证据矩阵
RS0→RS1→RS2→RS3→RS4 5门
/task-ddd
DDD 领域驱动分析 — 聚合/实体/值对象/领域服务/聚合行为清单+路由建议
agent指令模板 Gate B-DDD
/task-bdd
BDD 行为驱动场景 — Gherkin Given/When/Then 格式，BDD验收场景编写
agent指令模板 Gate B-BDD
/task-tdd
TDD 测试驱动任务分解 — Red→Green→Refactor 循环，TASK-XXX 映射 REQ-XXX
agent指令模板 Gate B-TDD

工程操作（7）

/refactor
代码重构安全网 — R1定义边界→R2基线测试+覆盖率→R3执行重构→R4行为漂移检测→R5报告
R1→R2→R3→R4→R5 5门
/hotfix
紧急热修复 — H0紧急声明+审批→H1最小化修复→H2快速验证+回滚→H3事后审计
H0→H1→H2→H3 4门 紧急协议
/migrate
框架迁移 — M1验证规则→M2应用迁移→M3编译验证→M4自动修复Lint(≤2轮)
M1→M2→M3→M4 4门
/debug
调试诊断 — D0收集信息→D1最小复现→D2调试会话→D3交互诊断→D4报告+修复方案
D0→D1→D2→D3→D4 5门
/evaluate
技术评估 — E0定义标准+权重→E1生成原型→E2收集指标→E3多维度报告+推荐
E0→E1→E2→E3 4门
/simplify
代码简化 — S0五桶分析(死代码/重复/过度抽象/AI痕迹/复杂性)→S1并行简化→S2回归验证→S3报告。失败自动回滚
S0→S1→S2→S3 4门 自动回滚
/improve
度量驱动迭代改进 — IM0量化目标→IM1研究→IM2计划→IM3执行+benchmark→IM4评估→迭代/停止。支持IM4→IM1循环
IM0→IM1→IM2→IM3→IM4 5门 allow_jump
/trace
因果追踪 — T0问题框架→T1假设生成(2-5竞争假设)→T2证据收集→T3贝叶斯更新(>70%方案/<70%重假设)→T4解决方案
T0→T1→T2→T3→T4 5门 贝叶斯

审查与修复（3）

/review
只读代码审查 — 不修改文件。spawn 审查 Agent 并行收集 findings（每条含文件/行号/依据），产出审查报告+严重度分级
只读 并行审查
/review-fix
审查修复优化闭环 — ①初审(只读)②修复规划③执行修复④验证(质量重检)⑤复审。最多2轮
闭环 ≤2轮
/bug-fix
Bug修复闭环 — 浏览器复现→根因定位→修复→浏览器验证。涉及前端/交互类 Bug 的7步闭环
浏览器闭环 Playwright

使用推荐

� 新手入门路径： /auto 需求描述 → 自动路由 + 引导 → /review 审查产出 → /review-fix 修复问题 → /release 发布
场景推荐指令原因
日常开发（默认）/auto自动任务分类+路由，最省心
中大型功能/jarvis13 Gate 完整流程，质量保证最强
重构代码/refactorR1-R5 安全网，基线→验证→漂移检测
紧急修复线上故障/hotfixH0-H3 紧急协议，审批+回滚预案
代码质量清理/simplify五桶分析，失败自动回滚
排查Bug/debug 或 /trace/debug=交互式诊断，/trace=贝叶斯因果推理
性能优化/improve度量驱动迭代，IM4→IM1循环直到达标
技术选型/evaluate 或 /research/evaluate=原型+指标，/research=深度分析+假设验证
需求不清晰/ask (Interview模式)Socratic追问→K1探索→K2评分矩阵
框架升级/migrateM1-M4 迁移+自动修复
代码审查/review → /review-fix先审查→后修复→再验证，最多2轮
测试补齐/test-unit / /test-integration按需选择测试类型
发布上线/release质量门+版本递增+CI自动发布

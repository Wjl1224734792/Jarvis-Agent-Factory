# `/test-perf` — 性能测试

- **命令**：`/test-perf [目标服务/接口]`
- **类别**：测试
- **说明**：定义 P50/P95/P99 延迟、吞吐量和错误率等性能目标，选择 k6/Artillery 等工具编写负载测试脚本，建立基线后执行多级并发场景，不达标时定位瓶颈并重测。

## 使用场景

| 场景 | 说明 |
|------|------|
| 接口压测 | 对关键 API 进行高并发压力测试，验证吞吐量和延迟 |
| 性能基线建立 | 为服务建立性能基线，后续版本对比退化情况 |
| 瓶颈定位 | 定位 DB、外部调用、序列化、内存等层面的性能瓶颈 |
| 容量规划 | 通过负载测试评估系统容量和扩展需求 |

## 关键 Agent

| Agent | 职责 |
|-------|------|
| perf-test-expert | 设计性能测试方案，编写和执行负载测试脚本 |
| perf-review-expert | 分析性能数据，定位瓶颈并给出优化建议 |

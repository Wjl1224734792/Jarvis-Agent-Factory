# `/trace` — 调用链追踪分析

- **命令**：`/trace [异常描述或问题现象]`
- **类别**：调研
- **说明**：基于贝叶斯推理的因果追踪流程，通过假设生成、证据收集与概率更新逐步缩小根因范围，直至定位问题。

## 使用场景

| 场景 | 说明 |
|------|------|
| 生产异常排查 | 线上报错或性能劣化时，系统化定位根因而非凭经验猜测 |
| 竞态条件诊断 | 多线程/异步场景下的时序问题，通过假设-证伪循环定位 |
| 数据不一致溯源 | 数据库或缓存中的脏数据，追踪写入链路定位污染源 |
| 性能瓶颈分析 | 接口延迟突增时，沿调用链逐层排查耗时热点 |

## 关键 Agent

| Agent | 职责 |
|-------|------|
| `code-explore-expert` | 代码调用链分析，收集支持/反对各假设的代码证据 |
| `external-resource-expert` | 外部资源检索，查阅框架文档、已知 Issue 与社区讨论 |
| `algorithm-expert` | 算法与数据流分析，生成竞态假设并计算先验/后验概率 |

## 流程图

```mermaid
flowchart TD
    START([用户输入 /trace 异常描述]) --> JOIN[session_join<br/>pipeline_type: trace]
    JOIN --> T0[Gate T0: 问题框架]
    T0 --> T0A[症状记录+上下文收集+已知信息矩阵]
    T0A --> T1[Gate T1: 假设生成]
    T1 --> T1A[spawn algorithm-expert<br/>生成2-5个竞态假设<br/>含先验概率+证伪条件]
    T1A --> T2[Gate T2: 证据收集]
    T2 --> T2A[spawn code-explore-expert<br/>并行收集支持/反对证据]
    T2A --> T2B[证据矩阵: 质量评估+后验概率]
    T2B --> T3[Gate T3: 因果分析]
    T3 --> T3A[贝叶斯更新: P(H|E)计算]
    T3A --> T3B{根因概率?}
    T3B --> |>70%| T4[Gate T4: 解决方案]
    T3B --> |40-70%| T2
    T3B --> |<40%| T1
    T4 --> T4A[修复方案+验证步骤+预防建议]
    T4A --> DONE([✅ 根因报告+修复方案])
```

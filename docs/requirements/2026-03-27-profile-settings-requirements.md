# 个人中心与设置页需求收敛
日期：2026-03-27
适用范围：`apps/web`

## 1. 需求摘要
在 `apps/web` 中完成个人中心页和设置页的设计与开发，要求与现有浅蓝航空主题统一，并接入真实路由和导航，形成前端可运行的闭环。

## 2. 目标与成功标准
- 个人中心页可从应用内入口访问。
- 设置页可从应用内入口访问。
- 页面风格与现有 `site-shell`、`Card`、`Button`、`Tabs`、`Input` 体系一致。
- 页面在未实现后端 profile/settings 接口的前提下，仍能提供可交互、可浏览、可回退的前端闭环。
- 现有登录态、注销和通知能力保持不破坏。

## 3. 范围内
- 路由接入。
- 导航入口接入。
- 个人中心页视觉与布局重构。
- 设置页视觉与布局重构。
- 前端态交互与表单状态整理。

## 4. 范围外
- 新增 profile/settings 后端接口。
- 账号体系重构。
- 管理后台开发。
- 共享契约大改。

## 5. 关键对象与交互
- 登录态用户信息来自 `useBootstrapAuth()` 和 `useAuthStore`。
- 个人中心和设置页复用 `site-shell` 布局和现有 UI 组件。
- 入口优先从现有用户菜单和壳层导航进入。

## 6. 主要风险
- 目前没有 profile/settings 后端接口，必须避免假装已有服务端能力。
- `apps/web/src/app.tsx`、`apps/web/src/features/auth/*`、`apps/web/src/routes/*` 属于同一前端壳层，改动要保持单线程，避免并发抢改。
- 路由与导航属于共享入口，变更要避免破坏现有首页、登录和发布流。

## 7. 推荐下一步
将该需求交给 `planner` 拆成可执行任务，并按前端实现顺序落地。

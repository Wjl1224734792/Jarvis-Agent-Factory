# Web 紧凑布局、资料持久化、评分清理与 admin 路由修复需求

## 1. 需求来源
- 本轮主会话已确认需求。

## 2. 需求摘要
- Web 端消息、个人中心、设置页改为紧凑布局，主内容区必须有明确 `min/max width`，禁止满屏大卡片。
- 页面中的“本地设置 / 后续接接口 / 开发中 / 占位 / 演练”等说明文案全部删除。
- 设置页不需要“常驻机场”字段。
- 个人资料至少支持头像、显示名、简介的真实持久化修改。
- 飞友圈、飞行器库、榜单页骨架屏需跟真实列数动态同步，不能写死列数。
- 机型卡片、机型详情页取消星星和评分，机型详情页改成正常评论区；评分相关数据模型一并去除。
- 榜单分类改为“热门 / 最新”，官方榜单不再单独分类，只显示官方标签。
- 榜单条目详情不再展示评分和星星。
- 发布飞行器页封面只能图片或视频二选一，不再同时存在封面图、图库和视频。
- 发布文章页补独立封面图片上传。
- 创建榜单页补飞行器库搜索与过滤，并改成紧凑编辑布局。
- admin 当前根路由需修复，不能再落到 React Router 默认 404，且要有兜底与更好的错误体验。

# 飞加 H5 / 微信小程序移动端前端需求文档

> 文档版本：v1.0  
> 更新时间：2026-04-15  
> 适用范围：移动端 H5、微信小程序  
> 接口基准：`http://localhost:3002/docs`、`http://localhost:3002/openapi.json`

## 1. 文档定位

本文档用于指导独立前端团队开发飞加的移动端 H5 与微信小程序用户端。

文档目标不是复制当前 Web 页面，而是基于仓库内已落地的用户侧能力和后端接口，整理出一套更适合移动端浏览、互动、发布的产品需求与 UI / UX 规范。

本文件只描述：

- 页面布局
- 功能逻辑
- 样式方向
- 交互规则
- 长列表形态
- 已有后端接口的复用边界

本文件不包含技术栈选型，不限定具体框架与组件库。

### 2.3 接口复用原则

- 以前端当前可访问的 OpenAPI 为唯一接口事实来源。
- 默认不新增后端接口，不新增前端私有协议，不绕开现有共享字段。
- 如果页面设计与接口字段不一致，先记录差异，再决定前端适配或后端补充。
- 所有公共内容、互动、发布、通知、用户资料等能力，均优先复用现有后端接口。

## 4. 移动端设计总原则

### 4.1 不复用 Web 布局

移动端不得直接照搬以下 Web 形态：

- 首页右侧边栏
- 飞行器库的桌面侧边筛选栏
- 飞友圈的弹窗式详情
- 发布页的左右分栏编辑 + 预览
- 搜索页的大块说明卡片 + 重复搜索框
- 详情页中的桌面留白结构和宽幅布局

移动端必须改为：

- 单列主内容结构
- 顶部导航 + 页面滚动内容区
- 一级页面底部导航
- 二级页面独立详情页
- 筛选使用顶部横向筛选条或底部弹层
- 发布页使用单列表单与底部固定操作栏

### 4.2 核心体验目标

- 一手可操作
- 内容优先
- 列表滚动顺畅
- 互动反馈及时
- 游客可浏览，登录只拦截受限动作
- H5 与微信小程序保持统一的信息架构和视觉语言

### 4.3 风格关键词

- 轻内容社区
- 航空感、但不过度拟物
- 干净、利落、信息密度适中
- 卡片层级明确
- 更像移动端原生内容产品，而不是压缩版 Web

## 6. 全局布局与样式规范

### 6.1 页面结构

一级页面统一采用：

1. 顶部安全区
2. 顶部导航栏
3. 页面内容滚动区
4. 底部导航栏

二级页面统一采用：

1. 顶部安全区
2. 返回导航栏
3. 页面内容滚动区
4. 底部固定操作栏或评论栏（按场景出现）

### 6.1.1 页面壳类型

移动端页面统一分为 4 类壳层：

| 壳层类型 | 适用页面 | 结构要求 |
| --- | --- | --- |
| 一级列表页 | 首页、飞友圈、榜单、我的 | 顶部导航 + 内容滚动区 + 底部 Tab |
| 二级详情页 | 帖子详情、飞行器详情、榜单详情、排行对象详情 | 返回导航 + 内容滚动区 + 底部操作栏 |
| 筛选 / 选择页 | 搜索结果、飞行器筛选、机型选择器 | 顶部搜索或筛选条 + 独立滚动区 |
| 发布页 | 各类发布页 | 返回导航 + 单列表单 + 底部固定提交栏 |

### 6.1.2 安全区与宽度

- 顶部必须适配状态栏安全区。
- 底部必须适配 Home Indicator 与小程序安全区。
- H5 在大屏手机与移动浏览器宽屏环境下，主体内容建议设置最大内容宽度并居中，避免无限拉宽。
- 微信小程序优先跟随宿主安全区，不自定义覆盖原生返回区域。

### 6.1.3 键盘与输入

- 带输入框页面必须处理键盘顶起与底部操作栏遮挡问题。
- 评论输入、登录验证码输入、发布表单输入时，焦点切换不能导致页面严重跳动。
- 底部评论栏在键盘弹出时应上移，键盘收起后恢复。

### 6.2 视觉规范

建议整体风格如下：

- 页面背景：浅灰或浅雾色
- 卡片背景：纯白
- 主文字：深灰接近黑色
- 次文字：中性灰
- 主色：偏天空蓝或深海蓝
- 强调色：用于评分、热度、收藏等

建议取值：

| 类型 | 建议值 |
| --- | --- |
| 页面背景 | `#F5F7FA` |
| 卡片背景 | `#FFFFFF` |
| 主文字 | `#18222D` |
| 次文字 | `#6B7280` |
| 主色 | `#1677FF` |
| 强调色 | `#F28C28` |
| 成功色 | `#16A34A` |
| 错误色 | `#DC2626` |

### 6.3 字号与圆角

| 场景 | 建议 |
| --- | --- |
| 页面主标题 | 18 px - 20 px |
| 卡片标题 | 15 px - 17 px |
| 正文 | 14 px - 15 px |
| 辅助信息 | 12 px - 13 px |
| 页面左右边距 | 12 px - 16 px |
| 卡片圆角 | 12 px - 16 px |
| 底部弹层顶部圆角 | 20 px - 24 px |

### 6.4 交互控件要求

- 点击热区不小于 44 px
- 横向滚动区域必须有明显的滚动提示或分页指示
- 所有列表页必须有骨架屏、空状态、错误状态、重试能力
- 所有长列表必须保留滚动位置
- 二级页默认保留返回按钮，并支持右上角分享或更多操作
- 长文页、详情页、发布页不出现桌面式按钮群

## 7. 全局交互规则

### 7.1 游客浏览与登录拦截

游客默认可浏览：

- 首页
- 飞友圈
- 飞行器库
- 榜单
- 搜索结果
- 内容详情
- 飞行器详情
- 榜单详情
- 排行对象详情
- 他人主页公开内容

以下动作必须登录后才能使用：

- 点赞
- 收藏
- 评论
- 回复
- 关注
- 举报
- 发布
- 查看我的私有页面
- 通知操作
- 编辑资料

未登录用户触发受限操作时：

- 优先弹出登录引导弹层
- 登录成功后返回原页面
- 尽量恢复原上下文和原操作

### 7.2 列表刷新与分页

所有列表页必须支持：

- 下拉刷新
- 上拉加载更多
- 首屏骨架屏
- 空状态
- 错误重试

所有长列表必须使用虚拟列表或等效的虚拟渲染能力，避免一次性渲染大量节点。

### 7.2.1 统一列表状态机

所有长列表统一遵循以下状态：

- `initial-loading`：首次加载，展示骨架屏
- `refreshing`：下拉刷新，保留旧数据并展示轻量刷新反馈
- `loading-more`：触底加载更多
- `empty`：成功但无数据
- `error`：请求失败
- `done`：已加载完成且无更多数据

统一要求：

- 必须有 `hasMore` 语义
- 必须有重复请求取消机制
- Tab 切换后返回时保留上次列表数据与滚动位置
- 从详情返回列表时恢复到原位置
- 弱网失败时允许用户原地点击重试

### 7.2.2 必须虚拟化的列表

- 首页主信息流
- 飞友圈瀑布流
- 飞行器库瀑布流
- 榜单长列表
- 通知列表
- 搜索结果长列表
- 创建榜单时的机型选择长列表

### 7.2.3 可不强制虚拟化但必须分页的区域

- 评论区根评论列表
- 回复列表
- 我的页最近内容列表
- 他人主页内容列表

说明：

- 评论树第一期可使用分页 + 按需展开回复，不要求完整树一次性渲染。
- 若目标端对动态高度瀑布流虚拟化支持不稳定，则允许退化为“分页加载 + 局部窗口渲染”，但不得一次性加载全部内容。

### 7.3 媒体展示规则

- 内容有视频时，优先展示视频区域。
- 内容无视频但有图片时，展示图片区域。
- 内容无视频、无图片时，展示纯文本卡片。
- 列表中的多图必须支持左右滑动查看，且不要求先进入详情页。
- 横向滑图手势优先级高于进入详情页手势。
- 列表中的视频允许直接显示视频播放器或可播放视频卡片，但默认不自动播放。
- 视频滚出可视区域后应自动暂停。

### 7.3.1 媒体卡片统一规则

| 场景 | 标题 / 摘要 | 媒体区 | 状态要求 |
| --- | --- | --- | --- |
| 首页卡片 | 标题最多 2 行，摘要最多 3 行 | 视频或横滑图片 | 必须有加载态、失败态、分页指示 |
| 飞友圈卡片 | 标题或摘要最多 2 行 | 封面图或视频首帧 | 必须有固定占位比例与播放角标 |
| 飞行器卡片 | 名称最多 2 行，摘要最多 2 行 | 机型封面 | 必须优先保证图片稳定加载 |
| 榜单卡片 | 标题最多 2 行 | 榜单预览条目图 | 无独立媒体轮播 |

统一要求：

- 图片加载失败要有兜底占位，不出现空白洞。
- 视频卡片需展示播放角标、时长或视频语义。
- 多图卡片需展示数量提示。
- 骨架屏结构应与真实卡片结构对应，避免切换时布局突变。

### 7.4 评论区规则

帖子、飞行器评测、排行对象评论区统一支持：

- 最新
- 热门

统一支持：

- 发表评论
- 回复评论
- 点赞评论
- 举报评论
- 作者编辑 / 删除自己的评论

### 7.4.1 导航与返回

- 一级页切换 Tab 时保留各自滚动位置与筛选状态。
- 二级页统一左上返回。
- H5 在浏览器返回不可用时，提供页面内兜底返回逻辑。
- 微信小程序详情页层级要尽量控制在可管理范围内，避免过深跳转。

### 7.5 分享规则

- H5 使用系统分享能力或复制链接。
- 微信小程序使用小程序分享能力。
- 分享文案优先使用标题，其次使用摘要。

### 7.6 状态提示

所有页面统一要求：

- 加载态不白屏
- 空状态有行动引导
- 错误态说明清楚，附带重试
- 上传失败支持重试
- 发表失败保留草稿

## 8. 四类核心长列表规范

## 8.1 首页信息流

### 定位

首页是内容消费主入口，信息流必须采用类 X 的单列时间线布局，不使用当前 Web 首页的桌面文章列表样式，也不使用双列瀑布流。

### 列表形态

- 单列长列表
- 使用虚拟列表
- 列表项支持文章与动态混排
- 推荐 / 最新 / 关注为主频道
- 可在主频道下增加横向内容分类 chips

### 单卡布局

```text
[头像] 昵称 / 认证 / 发布时间
       标题或正文摘要
       [视频播放器]
       或
       [图片横向滑动区 1 / N]
       或
       [纯文本，无媒体]
       评论  分享  点赞  收藏 / 浏览
```

### 卡片要求

- 头像、昵称、时间放在卡片顶部
- 标题与摘要按内容类型灵活展示
- 动态可弱化标题，突出正文摘要
- 文章可保留标题 + 摘要
- 媒体区位于正文下方、互动栏上方
- 互动栏固定在卡片底部

### 媒体交互

- 图片可左右滑动
- 支持页码点或 `1 / N`
- 横向滑动时不进入详情
- 点击图片可进入全屏预览
- 有视频时直接显示视频区域，不降级成普通封面图

### 与当前 Web 的差异

- 不保留首页右侧热门榜单、热门机型侧栏
- 热门榜单和热门机型改为首页流内模块或首页顶部快捷模块
- 不使用桌面两栏式文章卡片

### 推荐首页布局

```text
顶部导航
搜索入口
推荐 / 最新 / 关注
横向内容分类
首页单列信息流
流内穿插热门榜单模块
流内穿插热门机型模块
```

## 8.2 飞友圈列表

### 定位

飞友圈强调“社区动态 + 轻浏览 + 快互动”，列表必须采用双列瀑布流。

### 列表形态

- 双列瀑布流
- 使用虚拟列表
- 卡片高度自适应
- 推荐 / 最新 / 关注三类频道

### 卡片布局

```text
[封面图 / 视频首帧]
标题或前两行摘要
头像 昵称
点赞数 / 评论数
```

### 规则

- 视频卡片列表内不自动播放，可显示播放标识
- 点击卡片进入独立详情页
- 移动端不使用当前 Web 的弹窗详情模式
- 图片必须有占位比例，避免瀑布流跳动

## 8.3 飞行器库列表

### 定位

飞行器库是查询型内容入口，强调封面识别、机型信息扫读和快速筛选。

### 列表形态

- 双列瀑布流
- 使用虚拟列表
- 顶部搜索
- 顶部横向筛选 chips

## 9.25 页面骨架屏规范

### 9.25.1 骨架屏总原则

- 骨架屏必须尽量贴近真实页面结构，不得使用大面积统一灰块代替
- 骨架屏的层级、行数、媒体比例应与真实页面一致
- 列表页骨架屏应至少覆盖首屏可视区域
- 局部刷新优先使用“旧数据保留 + 局部骨架”，而不是整页闪白
- 骨架屏应区分首次加载、分页加载、局部刷新三种状态

### 9.25.2 启动与登录相关骨架

| 页面 | 骨架要求 |
| --- | --- |
| 启动页 | 不使用常规骨架，直接展示品牌启动视觉与轻量加载态 |
| 登录页 | 验证码区域可用局部骨架，占位手机号、验证码、按钮区域 |
| 注册补全页 | 头像区、昵称输入区、提交按钮使用轻量表单骨架 |

### 9.25.3 首页骨架

首页骨架必须包含：

- 顶部搜索入口占位
- 频道 Tab 占位
- 横向内容分类占位
- 3 到 4 条类 X 信息流卡片占位
- 流内热门榜单模块占位
- 流内热门机型模块占位

单条首页卡片骨架建议结构：


## 10. H5 与微信小程序接口口径

### 10.1 H5 登录口径

H5 默认复用 Web 侧登录接口：

- `POST /auth/captcha/challenge`
- `POST /auth/sms/request`
- `POST /auth/web/login`
- `POST /auth/web/register/complete`
- `POST /auth/web/refresh`
- `GET /auth/me`
- `POST /auth/logout`

### 10.2 微信小程序登录口径

微信小程序默认复用 App 侧登录接口：

- `POST /auth/captcha/challenge`
- `POST /auth/sms/request`
- `POST /auth/app/login`
- `POST /auth/app/register/complete`
- `POST /auth/app/refresh`
- `GET /auth/app/me`
- `POST /auth/app/logout`

说明：

- 业务内容接口、互动接口、发布接口尽量共用。
- 仅鉴权口径按端区分。

### 10.3 共用差异约束

前端团队需要额外统一以下差异口径：

- 分享落地页：分享后的页面默认支持游客态打开
- 路由参数：详情页参数规则在 H5 与微信小程序中保持一致
- 登录恢复：登录成功后尽量回到发起动作的页面
- Token / 会话刷新：避免并发刷新造成重复请求
- 图片预览、视频播放、文件选择：按平台能力差异做适配，但不改变页面结构
- H5 浏览器返回、小程序页面栈返回均需有兜底

## 11. 明确禁止照搬的现有 Web 方案

以下现有 Web 体验在移动端必须改造：

| Web 现状 | 移动端改造要求 |
| --- | --- |
| 首页右侧热门榜单 / 热门机型侧栏 | 改为首页顶部快捷模块或流内模块 |
| 飞友圈弹窗详情 | 改为独立详情页 |
| 飞行器右侧固定筛选栏 | 改为顶部筛选 + 底部弹层 |
| 发布页左右分栏 + 实时预览 | 改为单列表单 + 底部固定提交 |
| 搜索页正文重复搜索框 | 顶部导航栏保留唯一搜索框 |
| 桌面式宽详情页 | 改为窄屏纵向详情结构 |

## 13. 验收标准

### 13.1 功能验收

- H5 与微信小程序均可游客浏览公开内容
- 首页为类 X 的单列信息流
- 首页列表中多图可左右滑动查看，不进入详情页也能浏览图片
- 首页列表中有视频时直接展示视频区域
- 飞友圈与飞行器库为双列瀑布流
- 榜单为单列卡片长列表
- 四类长列表均使用虚拟列表或等效能力
- 搜索页只使用顶部导航搜索框作为唯一搜索入口
- 互动操作在未登录时统一弹登录引导
- 发布能力复用现有后端接口，不新增私有协议

### 13.2 布局验收

- 不出现 Web 式侧边栏
- 不出现发布页双栏编辑布局
- 不出现飞友圈弹窗式详情
- 所有二级页面为独立移动端详情页
- 各发布页均采用单列表单 + 底部固定操作栏

### 13.3 体验验收

- 列表滚动顺畅，无明显掉帧
- 图片加载前有占位，避免列表跳动
- 互动反馈及时，不刷新整页
- 列表返回后保留滚动位置
- 发表失败保留草稿
- H5 与微信小程序的分享、返回、登录恢复逻辑一致
- 首页、列表页、详情页、发布页、我的页均有与真实结构对应的骨架屏

## 14. 交付给前端团队的使用方式

前端团队应以本文档作为产品和交互基准，并同时参考以下资料：

- OpenAPI 文档：`http://localhost:3002/docs`
- OpenAPI JSON：`http://localhost:3002/openapi.json`
- 当前 Web 用户端页面与路由，作为业务能力参考，而不是布局参考

若实际开发中发现以下问题，需要先回传确认：

- 页面需要的字段在 OpenAPI 中不存在
- 某个流程无法通过现有接口闭环
- H5 与微信小程序的鉴权口径需要统一
- 某类长列表的数据量、排序或分页方式与当前接口不匹配

## 15. 结论

本期移动端 H5 与微信小程序的目标不是“把 Web 缩到手机里”，而是基于现有内容、社区、机型、榜单、用户、发布能力，重组为更适合移动端的用户体验：

- 首页采用类 X 的单列信息流
- 飞友圈与飞行器库采用双列瀑布流
- 榜单采用单列结构化卡片流
- 所有长列表采用虚拟列表
- 所有详情页采用独立移动端页面
- 所有发布页采用单列表单
- 所有业务能力优先复用现有后端接口

# 飞加微信小程序用户端需求文档

> 文档版本：v2.0  
> 更新时间：2026-04-15  
> 适用范围：微信小程序用户端  
> 技术选型：微信原生小程序 + TypeScript + TDesign 小程序版  
> 登录方式：微信一键登录，不使用手机号短信验证码  
> 接口基准：`http://localhost:17382/docs`、`http://localhost:17382/openapi.json`

## 1. 文档定位

本文档用于指导独立前端团队开发「飞加」微信小程序用户端。

小程序工程不在当前仓库内开发，不恢复 `apps/mobiles`，也不在本仓库新增小程序壳工程。当前仓库只作为需求、接口、业务规则和后端能力来源。小程序建议独立建仓或独立目录管理，并通过 OpenAPI 与 `apps/server` 对接。

本版本替换旧文档中的 H5 / 小程序双端口径，聚焦微信小程序单端开发。

## 2. 核心结论

- 只做微信小程序用户端，不包含 H5。
- 使用微信原生小程序开发，不使用 UniApp、Taro、Flutter 或 WebView 套壳。
- 使用 TypeScript。
- 使用 TDesign 小程序版作为基础组件库。
- 登录使用微信一键登录，不使用手机号 + 图形验证码 + 短信验证码。
- 手机号绑定不是本期登录前置条件；如后续业务需要，可作为可选资料补充能力。
- 页面体验按小程序原生交互设计，不照搬 Web 桌面布局。

## 3. 产品目标

### 3.1 目标

- 让用户在微信小程序内完成公开内容浏览、社区互动、飞行器资料查询、榜单浏览、内容发布和个人中心管理。
- 通过微信一键登录降低登录门槛，优先支持游客浏览，登录只拦截互动、发布和个人私有能力。
- 基于当前后端能力复用已有内容、社区、机型、榜单、用户、通知与上传接口。

### 3.2 成功标准

- 用户首次打开小程序可直接进入游客态首页。
- 用户点击「微信一键登录」后可完成登录或注册补全，不出现短信验证码流程。
- 首页、飞友圈、飞行器库、榜单、我的 5 个一级入口可用。
- 公开内容可浏览，登录后可点赞、收藏、评论、关注、举报、发布。
- 长列表具备下拉刷新、上拉加载、骨架屏、空状态、错误重试和滚动位置恢复。
- 小程序 UI 使用 TDesign 小程序版组件，视觉风格与飞加品牌一致。

## 6. 微信一键登录需求

### 6.1 登录原则

- 不使用手机号短信验证码。
- 不展示图形验证码。
- 不要求用户输入手机号。
- 登录入口统一文案为「微信一键登录」。
- 游客可浏览公开内容，触发受限动作时再弹出登录引导。

### 6.2 登录流程

```text
用户点击「微信一键登录」
→ 前端调用 wx.login 获取 code
→ 前端将 code 发送给后端
→ 后端换取 openid / unionid / session_key
→ 后端查找或创建用户
→ 若资料完整：返回登录态，进入原页面
→ 若资料不完整：返回 registration_required，进入资料补全页
→ 补全昵称 / 头像后完成注册
```

### 6.3 推荐接口

当前 OpenAPI 中已有 App 登录口径，但旧文档依赖短信验证码。本期需要后端提供微信小程序专用登录接口。建议接口如下：

| 能力 | 建议接口 | 说明 |
| --- | --- | --- |
| 微信小程序登录 | `POST /auth/wechat-mini/login` | 入参为 `code`，返回登录态或注册补全状态 |
| 微信小程序注册补全 | `POST /auth/wechat-mini/register/complete` | 补全昵称、头像 |
| 获取当前用户 | `GET /auth/wechat-mini/me` | 恢复会话 |
| 刷新会话 | `POST /auth/wechat-mini/refresh` | 登录态续期 |
| 退出登录 | `POST /auth/wechat-mini/logout` | 清理会话 |

### 6.4 登录响应建议

登录成功：

```json
{
  "kind": "authenticated",
  "user": {
    "id": "user_xxx",
    "displayName": "飞友",
    "avatarUrl": null,
    "role": "user"
  }
}
```

需要补全资料：

```json
{
  "kind": "registration_required",
  "registrationToken": "token_xxx",
  "suggestedDisplayName": "飞友1234"
}
```

### 6.5 资料补全

资料补全页只要求：

- 昵称：必填。
- 头像：可选，可使用微信头像或后续上传。

不要求：

- 手机号。
- 短信验证码。
- 图形验证码。
- 真实姓名。

### 6.6 登录拦截

未登录用户触发以下动作时弹登录引导：

- 点赞
- 收藏
- 评论
- 回复
- 关注
- 举报
- 发布
- 查看通知
- 编辑资料
- 查看我的私有页面

登录成功后需要回到原页面，并尽量恢复原操作上下文。

## 8. 全局体验规范

### 8.1 页面壳

一级页面：

```text
顶部安全区
顶部导航 / 搜索入口
内容滚动区
底部 TabBar
```

二级页面：

```text
顶部安全区
返回导航栏
内容滚动区
底部操作栏或评论栏
```

发布页：

```text
返回导航栏
单列表单
媒体上传区
底部固定提交栏
```

### 8.2 视觉方向

| 类型 | 建议值 |
| --- | --- |
| 页面背景 | `#F5F7FA` |
| 卡片背景 | `#FFFFFF` |
| 主文字 | `#18222D` |
| 次文字 | `#6B7280` |
| 主色 | `#1677FF` |
| 强调色 | `#F28C28` |
| 成功色 | `#16A34A` |
| 错误色 | `#DC2626` |

风格关键词：

- 航空感
- 轻社区
- 内容优先
- 信息层级清晰
- 小程序原生手感

### 8.3 列表状态

所有列表统一支持：

- 首屏骨架屏
- 下拉刷新
- 上拉加载更多
- 空状态
- 错误重试
- 无更多提示
- 返回列表后恢复滚动位置

状态机：

| 状态 | 说明 |
| --- | --- |
| `initial-loading` | 首次加载，展示骨架屏 |
| `refreshing` | 下拉刷新，保留旧数据 |
| `loading-more` | 触底加载更多 |
| `empty` | 请求成功但无数据 |
| `error` | 请求失败，可重试 |
| `done` | 已无更多数据 |

### 8.4 媒体规则

- 有视频时优先展示视频。
- 无视频但有图片时展示图片。
- 无媒体时展示纯文本卡片。
- 多图支持横向滑动。
- 点击图片进入小程序图片预览。
- 视频默认不自动播放。
- 视频滚出可视区域后自动暂停。

### 8.5 分享规则

- 使用微信小程序原生分享能力。
- 支持分享首页、帖子详情、飞行器详情、榜单详情、排行对象详情和他人主页。
- 分享标题优先使用内容标题，其次使用摘要。
- 分享落地页必须支持游客态打开。

## 9. 核心页面需求

## 9.1 启动页

### 目标

完成会话恢复和基础初始化。

### 功能

- 读取本地登录态。
- 调用 `GET /auth/wechat-mini/me` 或等价接口恢复用户。
- 恢复成功进入首页登录态。
- 恢复失败进入首页游客态。
- 不强制跳登录页。

## 9.2 微信登录弹层

### 触发场景

- 用户主动点击「微信一键登录」。
- 游客触发受限操作。

### 页面内容

- 品牌 Logo。
- 登录权益说明。
- 「微信一键登录」主按钮。
- 用户协议与隐私政策勾选项。
- 取消按钮。

### 交互要求

- 未勾选协议时点击登录，提示先同意协议。
- 登录中按钮进入 loading。
- 登录失败展示 Toast，可重试。
- 登录成功关闭弹层并返回原页面。

## 9.3 注册资料补全页

### 字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 头像 | 否 | 可使用微信头像或默认头像 |
| 昵称 | 是 | 默认填入后端建议昵称，允许修改 |

### 规则

- 昵称冲突时展示字段级错误。
- 完成后进入首页或返回触发登录的页面。
- 不要求手机号。

## 9.4 首页

### 布局

```text
顶部导航
搜索入口
快捷入口：飞行器库 / 榜单 / 发布 / 我的收藏
推荐 / 最新 / 关注
横向内容分类
单列信息流
流内热门榜单模块
流内热门机型模块
```

### 功能

- 默认进入推荐。
- 关注频道未登录时弹登录引导。
- 支持进入文章、动态、飞行器、榜单。
- 返回首页时恢复频道、滚动位置和已加载数据。

### 接口

- `GET /home/feed`
- `GET /rankings`
- `GET /models`

## 9.5 搜索页

### 功能

- 顶部搜索框为唯一搜索入口。
- 输入少于 2 个字符不发请求。
- 展示搜索历史与热门词。
- 搜索结果按文章、动态、飞行器、榜单、排行对象、用户分组。

### 接口

- `GET /search?q=`

## 9.6 飞友圈

### 列表

- 双列瀑布流。

### 详情

- 展示媒体、作者、标题、正文、互动栏、评论区。
- 支持点赞、收藏、分享、举报、关注作者、评论、回复。

### 接口

- `GET /circle/feed`
- `GET /posts/{id}`
- `POST /posts/{id}/interactions/{type}`
- `POST /posts/{id}/comments`
- `POST /posts/{id}/report`

## 9.7 飞行器库

- 双列瀑布流。

### 详情

## 10. 上传需求

### 10.1 上传流程

```text
用户选择图片 / 视频
→ 小程序读取临时文件
→ 调用 POST /uploads/init 获取上传信息
→ 小程序上传文件
→ 调用 POST /uploads/complete 完成上传
→ 将 fileId 写入业务提交接口
```

### 10.2 上传场景

- 头像
- 文章图片 / 视频
- 动态图片 / 视频
- 飞行器封面 / 图册 / 视频
- 品牌 Logo
- 榜单封面
- 排行对象图片
- 举报证据图

### 10.3 交互要求

- 上传中显示进度或 loading。
- 上传失败可重试。
- 删除媒体需二次确认。
- 提交时若仍有文件上传中，按钮不可用。

### 11.2 空状态

空状态必须包含：

示例：

- 暂无搜索结果：引导更换关键词。
- 暂无评论：引导登录后发表评论。
- 暂无我的内容：引导进入发布中心。

## 12. 与现有后端的接口差异

### 12.1 必须新增或调整的后端能力

| 能力 | 当前问题 | 建议 |
| --- | --- | --- |
| 微信一键登录 | 旧文档使用短信登录，不符合本期要求 | 新增 `POST /auth/wechat-mini/login` |
| 微信注册补全 | 需要承接一键登录后的新用户资料补全 | 新增 `POST /auth/wechat-mini/register/complete` |
| 小程序会话恢复 | 需要区别 Web / Admin / App 会话 | 新增 `GET /auth/wechat-mini/me` 或复用明确的小程序 scope |
| 小程序退出登录 | 需要清理小程序会话 | 新增 `POST /auth/wechat-mini/logout` |

### 12.2 可直接复用的后端能力

- 内容流
- 飞友圈
- 帖子详情与互动
- 飞行器库
- 榜单与排行对象
- 评论
- 举报
- 通知
- 用户主页
- 发布
- 上传

### 12.3 风险

- 如果后端不新增微信小程序登录接口，小程序无法满足「不使用短信验证码」的登录要求。
- 如果继续复用旧 App 短信登录接口，会与本需求冲突。
- 小程序端无法直接使用浏览器 Cookie 语义，request client 需要明确会话存储方式。

## 13. 验收标准

### 13.1 功能验收

- 小程序可游客态进入首页。
- 登录流程只使用微信一键登录。
- 全流程不出现短信验证码、图形验证码、手机号必填。
- 登录后可完成互动、发布、通知和个人资料编辑。
- 退出登录后回到游客态。
- 分享落地页游客可打开。

### 13.2 UI 验收

- 使用 TDesign 小程序版组件。
- 一级页面底部 Tab 清晰稳定。
- 二级页面有明确返回导航。
- 发布页为单列表单 + 底部固定提交栏。
- 不出现 Web 侧边栏、桌面宽表格、双栏编辑器。

### 13.3 体验验收

- 长列表滚动顺畅。
- 下拉刷新和上拉加载可用。
- 首屏不白屏。
- 弱网失败可重试。
- 媒体上传失败可重试。
- 登录成功后能回到触发登录的页面。

### 13.4 技术验收

- TypeScript 严格模式下无类型错误。
- 页面不直接调用 `wx.request`，统一走 request client。
- API 错误统一处理。
- 登录态失效统一跳登录引导。
- TDesign 组件按需使用，不引入其他小程序 UI 库。

如果开发中发现以下问题，需要先回传确认：

- 微信审核要求与当前内容展示方式冲突。

## 16. 结论

本期目标是开发一个原生微信小程序用户端，而不是把 Web 端压缩到小程序里。

最终交付应满足：

- 微信原生小程序开发。
- TypeScript 编写。
- TDesign 小程序版组件体系。
- 微信一键登录。
- 不使用短信验证码。
- 游客可浏览，登录拦截受限动作。
- 复用现有后端业务接口。
- 小程序工程独立于当前仓库开发。

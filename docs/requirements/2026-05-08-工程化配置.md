# 项目工程化配置

> 日期：2026-05-08 | 前置版本：v3.23.1

## 背景

项目当前是纯 JavaScript ESM CLI 工具，缺少：
- **Lint 工具**：无代码风格检查，依赖人工审查
- **测试框架**：无自动化测试，验证依赖 `node --check` + 手动脚本
- **类型检查**：纯 JS 无类型约束，易出错
- **版本管理**：同时维护 Gitee + GitHub，用户决定只保留 GitHub

## 需求

### REQ-001：ESLint 代码规范检查
- 安装 ESLint + 推荐规则集
- 配置 `.eslintrc` 或 `eslint.config.js`（flat config）
- 添加 `npm run lint` 脚本
- 规则适配 Node.js ESM 项目风格

### REQ-002：Vitest 测试框架
- 安装 Vitest
- 配置 `vitest.config.js`
- 添加 `npm run test` 脚本
- 为核心模块（db.js, gates.js, routes.js）编写基础单元测试

### REQ-003：JSDoc 类型检查
- 安装 TypeScript（仅用 `tsc --checkJs`）
- 配置 `jsconfig.json` 或 `tsconfig.json`（`checkJs: true`, `allowJs: true`）
- 添加 `npm run typecheck` 脚本
- 核心函数添加 JSDoc `@param`/`@returns` 类型注解

### REQ-004：Git 远程仓库清理
- 移除 `origin` (Gitee) 远程引用
- 将 `github` 远程重命名为主 `origin`
- 更新 `package.json` 的 `repository` 字段指向 GitHub

### REQ-005：npm 脚本整合
- `package.json` scripts 增加：
  - `lint` — ESLint 检查
  - `lint:fix` — ESLint 自动修复
  - `test` — Vitest 运行
  - `typecheck` — tsc --checkJs
  - `check` — 一键运行 lint + typecheck + test（CI 入口）

## 验收标准

1. `npm run lint` 对现有代码通过（或仅有可控 warning）✅
2. `npm run test` 核心模块测试通过 ✅
3. `npm run typecheck` 通过（或仅有少量类型错误待逐步修复）✅
4. `git remote -v` 只显示 GitHub ✅
5. `npm run check` 一键运行全部检查通过 ✅

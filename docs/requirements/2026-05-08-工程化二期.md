# 工程化二期需求文档

> 状态: confirmed | 日期: 2026-05-08

## 背景

一期已完成 ESLint + Vitest + JSDoc type-check + Git 清理。二期在基础之上：
- 从 JS+JSDoc 全量迁移到 TypeScript 强类型
- 搭建 CI/CD 自动化流水线（GitHub Actions + Husky）
- 清理历史遗留的 token 脚本和 Gitee 相关代码
- 统一 agent effort 级别
- 规范临时文件目录

## REQ-001：TypeScript 全量迁移

**优先级**: P0

**描述**: 将 `src/**/*.js` 和 `tests/**/*.js` 重命名为 `.ts`，移除 JSDoc 类型注解改为 TypeScript 语法，配置 `tsconfig.json` 编译输出 `dist/` 目录。

**验收标准**:
- [ ] 所有 `src/**/*.js` 改为 `src/**/*.ts`
- [ ] 所有 `tests/**/*.js` 改为 `tests/**/*.ts`
- [ ] `jsconfig.json` 替换为 `tsconfig.json`（严格模式）
- [ ] `npm run build` 能通过 tsc 编译产出 `dist/`
- [ ] `npm run typecheck` 在 TS 严格模式下 0 error
- [ ] 现有 ESLint 配置适配 TypeScript（`@typescript-eslint`）
- [ ] `npm run lint` 0 error
- [ ] `npm run test` 全部 18 测试通过
- [ ] `bin` 入口调整为指向编译产物（如需）
- [ ] 发布 `files` 字段调整为包含 `dist/`

## REQ-002：CI/CD 自动化流水线

**优先级**: P0

**描述**: 搭建 GitHub Actions workflow + Husky git hooks 实现 push→lint→typecheck→test→build→release→publish 全自动化。

**GitHub Actions workflow**（`.github/workflows/ci.yml`）:
- [ ] `push` 到 `main` / PR 时触发：lint + typecheck + test
- [ ] `v*` tag push 时触发：build + GitHub Release + npm publish

**Husky git hooks**:
- [ ] `pre-commit`: lint-staged（只检查暂存文件）
- [ ] `pre-push`: npm run check（lint + typecheck + test）

**npm 自动发布**:
- [ ] 本地：依赖 `~/.npmrc`（`npm login` 一次），`release.sh` 直接 `npm publish`
- [ ] CI：GitHub Actions 通过 Repository Secret `NPM_TOKEN` 发布

## REQ-003：Agent Effort 统一（max→high）

**优先级**: P1

**描述**: 将所有 Claude 平台 agent 模板中 `defaultEffort` 或 `effort` 值为 `"max"` 的改为 `"high"`。

**范围**:

- `src/templates/platforms/claude/skills/*/SKILL.md` 中如有 `effort: max` → `effort: high`

**验收标准**:
- [ ] grep `effort.*max` 在模板目录中无匹配结果
- [ ] 所有模板文件中的 effort 值均为 `high` 或保持原值

## REQ-004：Token/脚本/Gitee 清理

**优先级**: P0

**描述**: 删除含真实 token 的 `.env`，清理 token 占位符，删除 Gitee 同步脚本，将 release 脚本改用 `gh` CLI。

**具体操作**:

| 文件 | 操作 |
|------|------|
| `.env` | **删除**（含真实 GITEE_TOKEN/GITHUB_TOKEN/NPM_TOKEN） |
| `.env.example` | 只保留 `PORT=` 注释，删除所有 token 段 |
| `scripts/sync-gitee-releases.js` | **删除** |
| `scripts/sync-github-releases.js` | 改为用 `gh release` CLI，不再依赖 GITHUB_TOKEN |
| `scripts/release.sh` | 删除 Gitee 步骤(4/5)；GitHub 步骤改用 `gh release create`；npm 发布移除 `$NPM_TOKEN` 检查 |
| `.github/workflows/release.yml` | 改为 GitHub URL + `gh` CLI |

**验收标准**:
- [ ] `.env` 已删除（确认不在 git 追踪中）
- [ ] grep `GITEE_TOKEN\|GITHUB_TOKEN\|NPM_TOKEN` 在 `scripts/` 和 `.env*` 中无匹配
- [ ] `scripts/sync-gitee-releases.js` 已删除
- [ ] `scripts/release.sh` 使用 `gh release create`，不含 token 引用
- [ ] `.github/workflows/release.yml` 引用 GitHub URL

## REQ-005：临时文件目录约束

**优先级**: P1

**描述**: 所有 agent 子任务产出的临时文件必须统一放到 `docs/tmp/` 目录，禁止散落在项目根目录或其他位置。

**实现方式**:
- 在 `AGENTS.md` 中添加临时文件目录约束规则
- 在各平台 agent 模板的全局指引中加入此约束
- `mkdir -p docs/tmp` 确保目录存在

**验收标准**:
- [ ] `docs/tmp/` 目录存在（带 `.gitkeep`）
- [ ] `AGENTS.md` 含临时文件目录规则
- [ ] `.gitignore` 含 `docs/tmp/*`（允许 `.gitkeep`）

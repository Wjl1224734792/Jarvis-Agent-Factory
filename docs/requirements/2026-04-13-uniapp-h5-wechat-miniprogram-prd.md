# UniApp + Tailwind CSS + uView Pro H5 / 微信小程序需求文档

> 文档版本：v1.2  
> 更新时间：2026-04-13  
> 适用范围：H5、微信小程序  
> 技术栈：UniApp、Vue 3、TypeScript、Tailwind CSS、uView Pro  
> API 文档入口：[http://localhost:3002/docs](http://localhost:3002/docs)

## 1. 文档目标

本文档用于指导基于 `UniApp + Tailwind CSS + uView Pro` 的移动端产品设计与开发，适用于 H5 与微信小程序双端。

本文档重点明确：

- 产品功能范围与游客 / 登录边界。
- 首页、飞友圈、飞行器库、榜单、搜索、详情、发布、我的等核心页面的布局与交互。
- 移动端页面样式风格与组件使用规则。
- API 对接边界与媒体字段消费方式。

本项目强调“移动端优先”，不是把 Web 页面压缩到手机里。移动端要围绕内容消费、轻互动、快速浏览和单手操作来设计。

## 2. 产品定位

产品定位为“飞友与飞行器内容社区”的移动端入口，核心体验是：

- 快速浏览推荐内容。
- 轻量互动。
- 快速搜索公开内容。
- 低门槛发布内容。
- 在不同内容类型之间切换，但保留一致的移动端体验。

## 3. API 与鉴权原则

### 3.1 API 文档

前端联调与接口字段定义，以本地 API 文档为准：

- [http://localhost:3002/docs](http://localhost:3002/docs)

若页面设计与 API 返回结构不一致，应先记录差异，再决定是前端适配还是后端补充，不允许前端凭空定义接口字段。

### 3.2 游客浏览原则

- H5 与微信小程序都必须支持游客浏览。
- 用户进入应用后，不强制先登录。
- 游客可直接浏览首页、飞友圈、飞行器库、榜单、搜索结果、内容详情、公开个人主页等公开内容。
- 游客浏览链路不得因为未登录而被提前打断。

### 3.3 需要登录后才能使用的功能

以下功能需要登录后才能使用：

- 点赞。
- 收藏。
- 评论。
- 回复。
- 关注。
- 举报。
- 发布内容。
- 编辑个人资料。
- 查看“我的”下的私有内容。
- 消息通知相关操作。

### 3.4 登录拦截要求

- 只有当用户触发受限功能时，才要求登录。
- 未登录用户点击受限功能时，优先弹出登录引导，而不是直接跳转空白页。
- 登录成功后尽量回到原页面，并恢复原来的浏览上下文。
- 启动页仅检查本地登录态，不得把游客统一重定向到登录页。

## 5. 技术方案约束

### 5.1 基础技术

- 框架使用 `UniApp + Vue 3 + TypeScript`。
- 样式体系使用 `Tailwind CSS` 作为原子类基础。
- 通用交互组件优先使用 `uView Pro`。
- 页面样式允许在 `uView Pro` 基础上定制，但不得退化成 Web 后台风格。

### 5.2 组件职责

- `uView Pro` 负责输入框、按钮、弹窗、Tab、Toast、Navbar、Tag、Popup、ActionSheet、Skeleton、LoadMore、Empty 等通用交互。
- `Tailwind CSS` 负责布局、间距、圆角、阴影、层级、字体与颜色 Token。
- 自定义组件负责首页 X 风格信息流、双列瀑布流卡片、榜单单列卡片、媒体轮播、作者信息、互动栏、空状态等产品组件。

### 5.3 设计实现原则

- 页面布局必须以手机竖屏为默认形态设计。
- 不允许出现 PC 式左右分栏、常驻侧栏筛选、复杂工作台式布局。
- 一级页面优先考虑底部 Tab 与顶部导航栏。
- 筛选、排序、更多操作统一使用顶部吸顶筛选条、底部弹层或抽屉。

## 6. 设计总原则

### 6.1 移动端优先

- 页面宽度按手机端单列内容区设计。
- 顶部区域简洁，优先保留标题、搜索、切换、返回、操作按钮。
- 底部使用 `tabBar` 承载一级导航。
- 页面主体以滚动内容区为核心。
- 可点击区域不小于 `44 px`。

### 6.2 内容优先

- 首屏优先看到内容，不堆大面积说明文字。
- 卡片封面、标题、作者、互动数据要让用户一眼扫到。
- 页面层级主要通过留白、色块、阴影来表达，不依赖密集边框。

### 6.3 多信息流形态并存

移动端不再要求“所有信息流都用双列瀑布流”，而是按内容场景区分：

- 首页：X 风格单列信息流。
- 飞友圈：双列瀑布流卡片。
- 飞行器库：双列瀑布流卡片。
- 榜单：单列卡片信息流。
- 搜索结果：按结果类型决定，内容类可双列，榜单类保持单列。

## 7. 信息架构

建议底部一级导航为 5 个入口：

| Tab | 名称 | 说明 |
| --- | --- | --- |
| 1 | 首页 | 推荐内容、热门专题、搜索入口 |
| 2 | 飞友圈 | 社区动态内容流 |
| 3 | 发布 | 发布图文、动态、榜单等 |
| 4 | 榜单 | 官方榜单、社区榜单、排行对象 |
| 5 | 我的 | 个人资料、收藏、历史发布、设置 |

补充说明：

- 飞行器库不单独占底部 Tab，可放在首页快捷入口、搜索结果、榜单关联跳转和我的常用入口中。
- 搜索使用独立页面，不单独占用底部 Tab。
- 个人主页从“我的”或作者头像点击进入。

## 9. 全局布局规范

### 9.1 页面基础结构

统一页面结构如下：

1. 顶部安全区。
2. 顶部导航栏。
3. 页面内容区。
4. 底部安全区或底部操作栏。
5. 一级页面显示底部 `tabBar`，二级页面隐藏 `tabBar`。

### 9.2 导航栏规范

- 一级页面标题简洁明确，如“首页”“飞友圈”“榜单”“我的”。
- 二级页面使用返回按钮 + 标题。
- 不做 Web 式面包屑导航。
- 搜索页与搜索结果页只使用顶部导航栏搜索栏作为唯一搜索输入入口。
- 搜索页正文区域不再重复放第二个搜索框。

### 9.3 内容区规范

- 页面左右内边距建议为 `12 px - 16 px`。
- 模块间距建议为 `12 px - 20 px`。
- 卡片圆角建议为 `12 px - 16 px`。

## 10. 信息流规范

### 10.1 首页信息流

首页信息流采用 X 风格单列时间线，不使用 Web 首页的文章列表布局，也不使用双列瀑布流。

每条卡片建议结构：

- 作者头像。
- 作者昵称。
- 发布时间。
- 正文摘要。
- 话题标签 / 链接文字。
- 多图横向轮播或视频卡片。
- 点赞、评论、收藏、分享等互动栏。

首页卡片要求：

- 风格接近 X 的移动端动态流。
- 一屏展示更少但更完整的信息。
- 交互栏固定在每条卡片底部，不做复杂悬浮操作。
- 如果内容没有图片，则不显示图片区域。
- 不使用默认占位图硬撑媒体区。

### 10.2 飞友圈信息流

飞友圈使用双列瀑布流卡片。

卡片建议结构：

- 封面图或视频首帧。
- 标题或内容前两行摘要。
- 作者头像与昵称。
- 点赞 / 评论等轻互动数据。

要求：

- 卡片高度允许自然变化。
- 图片必须有占位，避免抖动。
- 视频卡片只显示首帧和播放标识，列表内不自动播放。

### 10.3 飞行器库信息流

飞行器库使用双列瀑布流卡片。

卡片建议结构：

- 机型封面。
- 机型名称。
- 品牌名称 / Logo。
- 简短摘要。
- 收藏数、评论数等轻数据。

要求：

- 以图片和机型识别为主。
- 不使用 Web 页面那种左筛选 + 右列表布局。

### 10.4 榜单信息流

榜单使用单列卡片信息流。

榜单卡片布局要求：

- 与现有 Web 榜单卡片保持相同的信息结构。
- 显示标题、榜单类型、条目数、评论数、平均分。
- 展示前 2-3 个预览条目。
- 保持单列纵向排列，不做瀑布流。

### 10.5 搜索结果信息流

搜索结果页按结果类型展示：

- 文章 / 动态 / 飞行器等内容类结果：可使用双列或适合移动端的内容卡片。
- 榜单类结果：保持单列榜单卡片。
- 用户类结果：单列用户卡片。

搜索页正文要求：

- 不再放第二个搜索框。
- 不再放一整块解释型说明卡片。
- 顶部导航栏搜索栏是唯一搜索输入入口。
- 正文只负责展示搜索历史、热门搜索、结果列表、空状态、加载态、错误态。

## 11. 媒体规范

### 11.1 首页多图轮播

- 首页信息流支持显示多张图片。
- 多图内容在卡片内左右滑动轮播。
- 显示页码点或 `1 / N` 指示。
- 无图时不显示图片区域。

### 11.2 文章详情媒体区

- 文章详情顶部支持多图横向轮播。
- 若只有 1 张图片，则展示单图。
- 若没有图片，则不展示图片区域。
- 不使用默认占位图。

### 11.3 视频规则

- 首页和列表页的视频内容只展示首帧或视频卡片。
- 列表内不自动播放视频。
- 进入详情后再播放。

### 11.4 API 字段消费规则

移动端媒体展示优先消费现有接口字段：

- 首页 / 飞友圈：`GET /home/feed`、`GET /circle/feed` 返回的 `images[]`、`videos[]`
- 内容详情：`GET /posts/{id}` 返回的 `images[]`、`videos[]`

当前结论：

- 后端已有 `images[]` / `videos[]` 字段，可支持首页和详情多图轮播。
- 本期默认不新增数据库字段。
- 若实现时发现 OpenAPI 未完整暴露这些字段，再同步调整 `packages/schemas`、`packages/http-client`、`apps/server` 和 OpenAPI 文档。

## 12. 页面需求

### 12.1 启动页

功能要求：

- 展示品牌 Logo 与启动视觉。
- 检查本地登录态。
- 已登录用户恢复会话。
- 未登录用户以游客身份进入首页。

### 12.2 登录页

功能要求：

- 支持手机号登录。
- 支持验证码获取。
- 支持协议勾选。
- 登录成功后返回原目标页或进入首页。

### 12.3 首页推荐页

功能要求：

- 支持游客浏览。
- 展示推荐、最新、关注等频道。
- 展示首页主时间线。
- 主时间线使用 X 风格单列信息流。

页面布局：

- 顶部导航栏。
- 搜索入口。
- 频道切换 Tabs。
- 单列信息流。

### 12.4 飞友圈页

功能要求：

- 支持游客浏览。
- 展示社区动态内容流。
- 使用双列瀑布流卡片。

### 12.5 飞行器库页

功能要求：

- 支持游客浏览。
- 展示机型列表与筛选结果。
- 使用双列瀑布流卡片。

### 12.6 榜单页

功能要求：

- 支持游客浏览。
- 展示官方榜单与社区榜单。
- 使用单列榜单卡片信息流。

### 12.7 搜索页与搜索结果页

功能要求：

- 支持游客搜索公开内容。
- 顶部导航栏提供唯一搜索入口。
- 支持搜索历史、热门搜索、结果分类。

页面布局：

- 顶部导航栏搜索栏。
- 正文展示历史、热门词、结果列表。
- 不再在正文放第二个搜索框。

### 12.8 内容详情页

功能要求：

- 支持游客阅读公开内容。
- 文章顶部支持多图轮播。
- 点赞、收藏、评论、关注等操作未登录时触发登录拦截。

页面布局：

- 顶部返回栏。
- 顶部媒体轮播区。
- 标题与作者信息区。
- 正文内容区。
- 评论区。
- 底部互动栏。

### 12.9 发布中心与发布页

功能要求：

- 需要登录。
- 发布页使用移动端单列表单。
- 不允许左右分栏编辑 + 预览。

### 12.10 我的与个人主页

功能要求：

- 我的页未登录时显示游客态与登录引导。
- 个人主页支持游客查看公开内容。
- 我的页和个人主页中的内容类列表可按页面需求使用双列内容流。

### 12.11 消息页与设置页

功能要求：

- 需要登录。
- 使用单列分组列表。
- 不使用瀑布流布局。

## 15. 交互规范

### 15.1 列表页

- 支持下拉刷新。
- 支持分页加载。
- 支持骨架屏。
- 支持空状态。
- 支持失败重试。

### 15.2 互动行为

- 点赞、收藏、关注使用乐观更新。
- 评论提交成功后局部刷新。
- 未登录触发互动时，优先弹出登录引导弹层。

### 15.3 游客浏览与登录拦截

- 游客默认可直接进入首页、飞友圈、飞行器库、榜单、搜索、内容详情、公开主页等页面。
- 用户点击点赞、收藏、评论、关注、发布、消息等受限功能时，再触发登录拦截。
- 登录拦截优先使用底部弹层、居中弹窗或轻跳转，不粗暴打断浏览链路。

### 15.4 搜索交互

- 顶部导航栏搜索栏承担输入、提交、清空、返回。
- 搜索页正文不再重复放搜索输入组件。
- 输入少于 2 个字符时不发起搜索请求。

## 16. 性能要求

- 首页首屏内容在良好网络下尽快可见。
- 首页媒体轮播和瀑布流图片必须懒加载。
- 图片必须有占位，减少布局抖动。
- 频道切换保留滚动位置与缓存结果。
- 评论区、详情页、消息页按需加载。

## 18. 验收标准

### 18.1 功能验收

- H5 与微信小程序都支持游客浏览公开内容。
- 只有在触发点赞、收藏、评论、关注、发布、消息等受限功能时才要求登录。
- 首页使用 X 风格单列信息流。
- 飞友圈与飞行器库使用双列瀑布流卡片。
- 榜单使用单列卡片信息流。
- 搜索页只使用顶部导航栏搜索栏作为唯一搜索入口。
- 首页与文章详情的图片支持左右滑动轮播。
- 无图内容不展示图片区域。
- 接口字段、鉴权方式、错误码与接口路径以 [http://localhost:3002/docs](http://localhost:3002/docs) 为准。

### 18.2 布局验收

- 页面整体为移动端布局，不出现 PC 式左右分栏。
- 发布页、设置页、消息页等非内容流页面使用单列布局。
- 筛选使用顶部条或底部弹层，不使用侧边栏。

### 18.3 样式验收

- `uView Pro` 与 Tailwind CSS 组合后的视觉风格统一。
- 首页、飞友圈、飞行器库、榜单四类页面风格统一但信息流形态不同。
- 信息密度适合手机阅读与触控操作。

## 19. 总结

本需求文档的核心要求如下：

1. 必须使用 `UniApp + Tailwind CSS + uView Pro` 作为统一开发基础。
2. 必须坚持移动端优先设计，而不是把 Web 页面压缩到手机里。
3. 首页采用 X 风格单列信息流，飞友圈和飞行器库为双列瀑布流，榜单为单列卡片流。
4. 搜索页只使用顶部导航栏搜索栏作为唯一搜索输入入口。
5. 首页与文章详情支持多图左右滑动轮播，无图不显示图片区域。
6. H5 与微信小程序都支持游客浏览，只有触发功能操作时才要求登录。
7. 接口实现与联调以 [http://localhost:3002/docs](http://localhost:3002/docs) 为准。

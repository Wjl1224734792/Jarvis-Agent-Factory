# UniApp + Tailwind CSS + uView Pro H5 / 微信小程序需求文档

> 文档版本：v1.1  
> 更新时间：2026-04-13  
> 适用范围：H5、微信小程序  
> 技术栈：UniApp、Vue 3、TypeScript、Tailwind CSS、uView Pro  
> API 文档入口：[http://localhost:3002/docs](http://localhost:3002/docs)

## 1. 文档目标

本文档用于指导基于 `UniApp + Tailwind CSS + uView Pro` 的移动端产品开发，覆盖 H5 与微信小程序双端。文档重点明确以下内容：

- 产品功能范围。
- 页面清单与页面布局。
- 移动端优先的交互方式。
- 视觉风格与组件使用规范。
- API 对接入口与鉴权边界。
- 列表、信息流、瀑布流、发布页、个人中心等核心页面的实现要求。

本项目不是把 Web 页面缩小后搬到手机端，而是重新按移动端内容消费与轻交互场景设计。所有内容流页面统一采用双列瀑布流，不使用 Web 常见的左右栏、侧边筛选栏或大表格布局。

## 2. 产品定位

产品定位为面向移动端用户的内容社区与轻内容发布平台，核心体验是“快速浏览、轻量互动、随手发布、持续消费”。

用户主要使用场景：

- 在碎片时间浏览推荐内容。
- 通过双列瀑布流快速扫视更多卡片。
- 点赞、收藏、评论、分享、关注。
- 发布图文、短内容、榜单或其他社区内容。
- 查看个人主页、消息通知、收藏内容与历史发布。

## 3. API 与鉴权原则

### 3.1 API 文档

接口字段、请求方式、鉴权方式、错误码、返回结构与联调说明，以本地 API 文档为准：

- [http://localhost:3002/docs](http://localhost:3002/docs)

前端开发不得凭空定义接口字段。若页面需求与 API 文档不一致，应先标记差异，再确认后端是否补充接口或调整前端展示。

### 3.2 游客浏览原则

- H5 与微信小程序都必须支持游客浏览。
- 用户进入应用后，不强制先登录。
- 游客可直接浏览首页、发现页、搜索结果页、内容详情页、专题页、公开个人主页等公开内容。
- 一级信息流与详情阅读链路不得因为未登录而被提前拦截。

### 3.3 需要登录后才能使用的功能

以下功能需要登录后才能使用：

- 点赞。
- 收藏。
- 评论。
- 回复。
- 关注。
- 举报。
- 发布内容。
- 编辑个人资料。
- 查看“我的”下的私有内容。
- 消息通知相关操作。

### 3.4 登录拦截要求

- 只有当用户触发受限功能时，才要求登录。
- 未登录用户点击受限功能时，优先弹出登录引导，而不是直接跳转空白页。
- 登录成功后尽量回到原页面，并恢复原来的浏览上下文。
- 启动页仅检查本地登录态，不得把游客统一重定向到登录页。

## 5. 技术方案约束

### 5.1 基础技术

- 框架使用 `UniApp + Vue 3 + TypeScript`。
- 样式体系使用 `Tailwind CSS` 作为原子类基础。
- 通用业务组件优先使用 `uView Pro`。
- 页面与业务模块样式允许在 `uView Pro` 基础上做二次定制，但不得回退成 Web 管理后台风格。

### 5.2 组件职责

- `uView Pro` 负责输入框、按钮、弹窗、Tab、Toast、Navbar、Tag、Popup、ActionSheet、Skeleton、LoadMore 等通用交互组件。
- `Tailwind CSS` 负责布局、间距、圆角、阴影、层级、栅格、字体大小、颜色变量映射。
- 自定义组件负责双列瀑布流卡片、内容卡片、作者信息、互动栏、空状态、上传卡片、发布表单块等产品化 UI。

### 5.3 设计实现原则

- 页面布局必须以手机竖屏为默认形态设计。
- 不允许出现 PC 式顶部导航 + 左侧菜单 + 右侧内容布局。
- 不允许把筛选条件常驻在页面左侧或右侧。
- 筛选、排序、更多操作统一使用顶部吸顶筛选条、底部弹层或抽屉。
- 所有信息流统一为双列瀑布流。

## 6. 设计总原则

### 6.1 移动端优先

- 页面宽度按手机端单列内容区设计。
- 顶部区域简洁，优先保留标题、搜索、切换、返回、操作按钮。
- 底部使用 `tabBar` 承载一级导航。
- 页面主体以滚动内容区为核心。
- 手指可点击区域不小于 44 px。

### 6.2 内容优先

- 首屏优先展示内容，而不是大面积说明文字。
- 卡片封面、标题、作者、互动数据要让用户一眼扫到。
- 页面层级用留白、阴影、色块区分，不依赖密集边框。

### 6.3 双列瀑布流优先

以下页面统一使用双列瀑布流：

- 首页推荐流。
- 发现页信息流。
- 分类内容流。
- 搜索结果中的内容流。
- 个人主页内容流。
- 我的收藏内容流。
- 话题或专题聚合流。
- 内容详情页底部相关推荐流。

如为评论列表、通知列表、设置列表、表单页，不使用瀑布流，保持单列。

## 7. 信息架构

| Tab | 名称 | 说明 |

| 1 | 首页 | 推荐内容、热门专题、搜索入口 |
| 2 | 发现 | 分类、话题、趋势内容流 |
| 3 | 发布 | 发布图文、动态、榜单等 |
| 4 | 消息 | 评论、点赞、关注、系统通知 |
| 5 | 我的 | 个人资料、收藏、历史发布、设置 |

补充说明：

- 搜索使用独立页面，不单独占用底部 Tab。
- 个人主页从“我的”或作者头像点击进入。
- 内容详情、评论回复、发布流程均为二级或三级页面。

## 9. 全局布局规范

### 9.1 页面基础结构

统一页面结构如下：

1. 顶部安全区。
2. 顶部导航栏。
3. 页面内容区。
4. 底部安全区或底部操作栏。
5. 一级页面显示底部 `tabBar`，二级页面隐藏 `tabBar`。

### 9.2 导航栏规范

- 一级页面标题简洁明确，如“首页”“发现”“消息”“我的”。
- 支持搜索入口、筛选入口、快捷操作入口。
- 二级页面使用返回按钮 + 标题。
- 不做 Web 式面包屑导航。

### 9.3 内容区规范

- 页面左右内边距建议为 `12 px - 16 px`。
- 模块间距建议为 `12 px - 20 px`。
- 卡片圆角建议为 `12 px - 16 px`。
- 大面积纯白背景时需加入浅灰底或分组底色，避免页面发飘。

## 10. 双列瀑布流规范

### 10.1 使用原则

- 所有信息流页面必须采用双列瀑布流。
- 卡片高度允许根据封面比例和摘要字数自然增长。
- 双列间距建议为 `10 px - 12 px`。
- 左右边距建议为 `12 px - 16 px`。
- 滑动过程中保持稳定，不允许图片加载后明显跳动。

### 10.2 卡片结构

瀑布流卡片统一包含：

- 封面图或视频首帧。
- 标题，最多显示 2 行。
- 摘要，可选，最多显示 2 行。
- 作者头像与昵称。
- 点赞数、评论数、收藏数中的 1-2 个关键指标。

### 10.3 图片比例

- 内容封面建议以 `3:4`、`4:5`、`1:1` 为主。
- 卡片图片不强制统一高度，但必须有预估占位高度。
- 首次渲染时先占位，再渐进加载图片。

### 10.4 交互规则

- 下拉刷新。
- 上拉分页加载。
- 懒加载图片。
- 首屏使用骨架屏。
- 切换 Tab 时保留各自滚动位置。
- 卡片点击进入详情页，不在列表内展开正文。

## 11. 页面需求

### 11.1 启动页

功能要求：

- 展示品牌 Logo 与启动视觉。
- 检查本地登录态。
- 自动恢复已登录用户状态。
- 未登录用户以游客身份进入首页。

页面布局：

- 全屏品牌背景。
- 居中 Logo 与一句品牌标语。
- 底部显示版本信息或轻提示。

样式风格：

- 简洁克制。
- 不堆砌按钮。
- 启动动画轻量，时长控制在 1 秒内。

### 11.2 登录页

功能要求：

- 支持手机号登录。
- 支持验证码获取。
- 支持勾选协议。
- 登录成功后返回原目标页或进入首页。

页面布局：

- 顶部品牌区。
- 中部表单区。
- 底部协议区与辅助说明。

样式风格：

- 表单块居中。
- 主按钮突出。
- 错误提示贴近字段展示。

### 11.3 首页推荐页

功能要求：

- 支持游客浏览。
- 展示推荐、最新、关注等频道。
- 展示搜索入口。
- 展示活动位或专题位。
- 展示推荐信息流。
- 关注频道未登录时可展示登录引导空状态，不能影响其他频道浏览。

页面布局：

- 顶部搜索框。
- 频道切换 Tabs。
- 可选专题横滑区。
- 双列瀑布流主内容区。

样式风格：

- 首页首屏优先看到内容，不做大 Banner 压屏。
- 专题区可用浅色块或横滑卡片增强节奏。
- 信息流卡片保持统一视觉语言。

### 11.4 发现页

功能要求：

- 支持游客浏览。
- 展示热门话题、分类、趋势内容。
- 支持按分类筛选内容流。

页面布局：

- 顶部分类切换。
- 横向滑动的话题标签区。
- 双列瀑布流内容区。

样式风格：

- 比首页更强调标签与主题聚合。
- 颜色可以略活跃，但不要杂乱。

### 11.5 搜索页与搜索结果页

功能要求：

- 支持游客搜索公开内容。
- 支持关键词搜索。
- 支持历史搜索。
- 支持热门搜索。
- 支持结果分类。

页面布局：

- 顶部搜索输入框。
- 搜索前展示历史与热门关键词。
- 搜索后展示分类切换与双列瀑布流结果。

样式风格：

- 搜索前页面偏轻量。
- 搜索后快速进入结果消费状态。

### 11.6 内容详情页

功能要求：

- 支持游客阅读公开内容。
- 展示封面、标题、作者、正文、标签、互动区。
- 支持点赞、收藏、评论、分享、关注作者。
- 点赞、收藏、评论、关注等操作在未登录时触发登录拦截。
- 支持相关推荐。

页面布局：

- 顶部返回栏。
- 头图或媒体区。
- 标题与作者信息区。
- 正文内容区。
- 标签区。
- 固定底部互动栏。
- 评论区。
- 底部相关推荐双列流。

样式风格：

- 正文阅读舒适，正文宽度适合手机阅读。
- 操作栏固定底部，方便单手操作。
- 相关推荐仍保持双列瀑布流风格。

### 11.7 发布中心页

功能要求：

- 提供多种发布入口。
- 展示发布规则、草稿状态、最近编辑。
- 未登录进入发布中心时触发登录引导。

页面布局：

- 顶部标题。
- 中部大按钮矩阵或卡片入口。
- 下方发布规则说明区。

样式风格：

- 发布入口要清晰、易点。
- 不做复杂仪表盘式布局。

### 11.8 发布图文页 / 发布动态页

功能要求：

- 需要登录。
- 支持标题、正文、图片、标签、话题。
- 支持草稿保存。
- 支持发布校验。

页面布局：

- 顶部导航栏带返回与发布按钮。
- 中部单列表单块。
- 图片上传区使用宫格或横向缩略图。
- 底部弹层用于标签、分类、话题选择。

样式风格：

- 发布页必须是移动端单列表单。
- 不允许左右分栏编辑 + 预览布局。
- 输入区、上传区、选择区分块清晰。

### 11.9 我的页面

功能要求：

- 未登录时展示游客态卡片和登录引导。
- 已登录时展示个人资料摘要。
- 展示收藏、我的发布、浏览记录、设置等入口。
- 展示我的内容流。

页面布局：

- 顶部个人信息卡。
- 功能入口宫格或列表。
- 我的内容 Tabs。
- 我的内容双列瀑布流。

样式风格：

- 顶部资料卡可以更有品牌感。
- 下方内容区保持简洁有序。

### 11.10 个人主页

功能要求：

- 支持游客查看公开个人主页。
- 展示用户头像、昵称、简介、关注关系。
- 展示用户公开内容流。
- 关注操作在未登录时触发登录拦截。

页面布局：

- 顶部用户信息区。
- 关注与私信等操作区。
- 内容分类 Tabs。
- 双列瀑布流内容区。

样式风格：

- 强调用户身份信息与内容输出。
- 内容区仍然优先于装饰性信息。

### 11.11 消息页

功能要求：

- 需要登录。
- 展示点赞、评论、关注、系统通知。
- 支持已读、未读状态。

页面布局：

- 顶部分类 Tabs。
- 单列消息列表。

样式风格：

- 消息页强调效率与可扫读性。
- 单条消息信息层级清晰。

### 11.12 设置页

功能要求：

- 需要登录。

样式风格：

- 简洁、稳定、原生感。
- 不做内容型瀑布流布局。

## 12. 视觉风格定义

### 12.1 风格关键词

- 轻内容。
- 年轻化。
- 有节奏感。
- 干净。
- 有品牌识别度。
- 偏移动端原生社区体验。

### 12.2 颜色建议

建议采用“浅底 + 高识别主色 + 少量强调色”的方式：

- 主背景：`#F6F7F9`
- 卡片背景：`#FFFFFF`
- 主文字：`#1F2937`
- 次文字：`#6B7280`
- 主色：`#1677FF`
- 强调色：`#FF6A3D`
- 成功色：`#16A34A`
- 错误色：`#EF4444`

要求：

- 主色用于主按钮、选中态、重要标签。
- 强调色用于热度、活动、推荐角标。
- 不使用高饱和大面积渐变作为常规背景。

### 12.3 字体与层级

- 页面标题：`18 px - 20 px`，字重 `600 - 700`
- 卡片标题：`15 px - 16 px`，字重 `600`
- 正文：`14 px - 15 px`
- 辅助信息：`12 px - 13 px`
- 标签与统计信息：`11 px - 12 px`

### 12.4 圆角与阴影

- 卡片圆角：`14 px`
- 按钮圆角：`10 px - 12 px`
- 输入框圆角：`10 px`
- 弹层圆角：顶部 `20 px`
- 阴影轻量，避免厚重悬浮感。

## 14. 交互规范

### 14.1 列表页

- 支持下拉刷新。
- 支持分页加载。
- 支持骨架屏。
- 支持空状态。
- 支持失败重试。

### 14.2 互动行为

- 点赞、收藏、关注使用乐观更新。
- 评论提交成功后局部刷新。
- 未登录触发互动时，优先弹出登录引导弹层。

### 14.3 游客浏览与登录拦截

- 游客默认可直接进入首页、发现、搜索、内容详情、公开主页等页面。
- 用户点击点赞、收藏、评论、关注、发布、消息等受限功能时，再触发登录拦截。
- 登录拦截优先使用底部弹层、居中弹窗或轻跳转，不要粗暴打断浏览链路。
- 登录完成后应返回触发前页面，并尽量保留当前滚动位置和浏览状态。

### 14.4 筛选与排序

- 顶部筛选条用于高频筛选。
- 复杂筛选项放入底部弹层。
- 排序切换不跳转新页面，优先局部刷新当前列表。

## 15. 性能要求

- 首页首屏内容在良好网络下尽快可见。
- 双列瀑布流图片必须懒加载。
- 图片必须有占位，减少布局抖动。
- 频道切换保留滚动位置与缓存结果。
- 评论区、详情页、消息页按需加载。

## 17. 验收标准

### 17.1 功能验收

- H5 与微信小程序功能结构一致。
- H5 与微信小程序都支持游客浏览公开内容。
- 只有在触发点赞、收藏、评论、关注、发布、消息等受限功能时才要求登录。
- 首页、发现页、搜索结果页、个人主页、收藏页等信息流全部为双列瀑布流。
- 登录、互动、发布、消息、设置流程可闭环完成。
- 接口字段、鉴权方式、错误码与接口路径以 API 文档 [http://localhost:3002/docs](http://localhost:3002/docs) 为准。

### 17.2 布局验收

- 页面整体为移动端布局，不出现 PC 式左右分栏。
- 发布页、设置页、消息页等非内容流页面使用单列布局。
- 筛选使用顶部条或底部弹层，不使用侧边栏。

### 17.3 样式验收

- 视觉风格统一。
- `uView Pro` 组件风格经过 Tailwind CSS 二次整理后保持一致。
- 卡片、按钮、输入框、弹层、标签风格统一。
- 信息密度适合手机阅读与触控操作。

## 18. 实施建议

- 先搭建全局主题变量、颜色 Token、间距 Token、圆角 Token。
- 优先封装 `WaterfallFeed`、`FeedCard`、`LoginRequiredSheet`，作为全站内容流和登录拦截基础组件。
- 再实现首页、发现页、搜索结果页与个人主页，统一验证双列瀑布流方案。
- 最后扩展发布、消息、设置等非瀑布流页面。

## 19. 总结

本需求文档的核心要求如下：

1. 必须使用 `UniApp + Tailwind CSS + uView Pro` 作为统一开发基础。
2. 必须坚持移动端优先设计，而不是把 Web 页面压缩到手机里。
3. 只要是信息流页面，统一采用双列瀑布流，形成一致的内容消费体验。
4. H5 和微信小程序都支持游客浏览，只有触发功能操作时才要求登录。
5. 接口实现与联调以 API 文档 [http://localhost:3002/docs](http://localhost:3002/docs) 为准。

# Web 详情页与发布页顶部导航栏需求

## 需求摘要

为 `apps/web` 的详情页与发布页补充统一的顶部导航栏体验。详情页需要接入与主站顶部导航栏一致的核心功能；发布页需要复用同一套顶部导航栏能力，但不显示搜索框。个人主页与他人主页同样纳入详情页范围，且顶部导航栏中的个人入口始终显示并跳转到“自己”的主页。

## 目标与成功标准

- 详情页新增顶部导航栏，交互能力与当前 `WebLayout` 顶部导航栏保持一致。
- 发布页新增顶部导航栏，保留导航、发布菜单、用户菜单等能力，但不显示搜索框。
- 顶部导航逻辑抽离为可复用组件，避免 `WebLayout` 与沉浸式页面重复维护。
- 用户访问自己主页或他人主页时，顶部导航栏中的个人入口文案与目标均保持为“自己”。
- 不改动后端接口、共享协议与路由常量语义。

## 范围内

- `apps/web` 内以下详情页接入顶部导航栏：
  - 机型详情页
  - 帖子详情页
  - 榜单详情页
  - 评分对象详情页
  - 自己主页
  - 他人主页
- `apps/web` 内以下发布页接入顶部导航栏：
  - 发布文章
  - 发布动态
  - 发布飞行器
  - 品牌申请
  - 榜单编辑
  - 发布结果页
- 抽取可复用顶部导航组件，并让现有主站布局复用该组件。
- 补充与顶部导航显示规则相关的前端测试。

## 范围外

- 不调整左侧桌面导航栏的信息架构。
- 不新增新的导航入口、路由或后端接口。
- 不改造 Admin 端导航。
- 不对页面主体内容、表单字段或详情页内容结构做顺手重构。

## 关键模块 / 功能列表

1. 顶部导航栏复用组件
2. 详情页顶部导航接入
3. 发布页顶部导航接入（禁用搜索框）
4. “自己”入口文案与跳转规则统一
5. 导航相关测试与回归验证

## 风险与开放问题

- 详情页与发布页当前使用 `ImmersiveLayout` / `PublishShell` 等不同壳层，接入公共导航时需要避免破坏现有间距与 sticky 行为。
- 复用现有 `WebLayout` 顶栏逻辑时，需要确保移动端抽屉、发布菜单与登录拦截行为不出现回归。
- 发布页禁用搜索框后，需明确仍保留品牌标识、发布入口与用户菜单，以满足“功能一致但不带搜索框”的要求。

## 已收敛结论

- 用户已确认详情页范围包含自己主页与他人主页。
- 用户已确认他人主页顶部导航栏中的个人入口应显示为“自己”。
- 本次实现仅针对 `apps/web`，不包含 `apps/admin` 与 `apps/server`。

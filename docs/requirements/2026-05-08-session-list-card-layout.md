# 需求文档：会话列表卡片化布局

> **状态**：confirmed | **日期**：2026-05-08 | **范围**：前端 UI 重构

## 背景

当前 Web 面板左侧侧边栏的会话列表，每个会话项采用**单行水平排列**：置顶标记 → 标题 → 指令标签 → Gate → 状态点 → ⋮ 菜单。所有信息挤在一行，在 288px 窄侧边栏中标题被严重截断，可读性差。

用户要求改为**紧凑的上下排列**（非大卡片），每项垂直展示 2-3 行信息，右侧始终可见 ⋮ 更多操作按钮。

## 需求

### REQ-SL-001：会话项垂直排列

会话列表中的每一项从单行水平排列改为紧凑的垂直排列（2 行或 3 行）：

- **第一行**（顶部）：任务标题（加粗，truncate）+ 右侧状态指示
- **第二行**（底部）：指令标签 + Gate 进度 + 平台标识 + ⋮ 更多按钮

**验收标准**：
- 每项高度不超过当前单行高度的 2.5 倍
- 标题在 288px 宽侧边栏中有至少 180px 的可视宽度
- 所有信息不再被挤到不可见的溢出区域

### REQ-SL-002：⋮ 更多菜单始终可见

当前 ⋮ 菜单依赖 `.session-actions { opacity: 0 }` hover 显示，且仅当有活跃 run 时才渲染。

改为：
- 每个会话项右侧始终渲染 ⋮ 按钮（使用 `more-vertical` 图标）
- 有活跃 run 时：点击展示下拉菜单（置顶/取消置顶、归档、删除）
- 无活跃 run 时：按钮显示但置灰不可点击，title 提示"暂无运行记录"

**验收标准**：
- 所有会话项都能看到 ⋮ 按钮（一致性）
- 无 run 的会话项按钮视觉上为灰色（`text-slate-300`），不可点击
- 有 run 的会话项按钮正常可点击，弹出菜单

### REQ-SL-003：选中态与悬停态

- 选中当前会话：左侧 border 高亮 + 背景色区分（保持现有 indigo 主题色）
- 悬停：轻微背景色变化 + ⋮ 按钮颜色加深
- 置顶会话：顶部 border 琥珀色标记（保持现有 pinned 视觉）

**验收标准**：
- 选中态与未选中态清晰可辨
- 不引入新的颜色变量，复用现有 Tailwind 主题色

### REQ-SL-004：保持现有功能不变

以下功能保持不变：
- 平台筛选（全部 / Claude / OpenCode / Codex）
- 置顶排序（pinned 排最前）
- 状态指示（在线 🟢 / 休眠 🟡 / 离线 ⚫）
- 会话选中切换
- 休眠会话恢复按钮
- 点击外部关闭下拉菜单
- 5 秒轮询刷新

**验收标准**：
- 所有现有交互功能不受影响
- 回归测试通过

## 不在范围

- 不修改归档面板布局
- 不修改智能体配置页
- 不修改主内容区的 Gate 状态追踪卡片
- 不修改历史 Runs 面板
- 不涉及后端 API 变更

## 技术约束

- 只修改 `src/web/views/pipeline.html` 中 `renderSessions()` 函数（约第 500-558 行）
- 只修改相关 CSS 样式（内联 `<style>` 或 Tailwind 类名）
- 不引入新依赖
- 不修改路由或数据流

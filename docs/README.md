# Docs — 流水线产物目录

> 每次提交代码后检查本文件是否需要同步更新目录结构。

## 阶段目录（按文档类型）

| 目录 | 对应 Gate | 内容 |
|------|----------|------|
| `requirements/` | Gate A | 需求文档 `REQ-XXX`，含架构决策记录（ADR） |
| `tasks/` | Gate B | 任务分解文档 `TASK-XXX`，DDD 分析 |
| `architecture/` | Gate B1 | 架构评审报告 |
| `plans/` | Gate C | 执行计划、并行批次、Execution Packets |
| `implementation/` | Gate C→C1 | 各 agent 实现完成报告 |
| `testing/` | Gate C2 | 测试用例、测试报告、覆盖率 |
| `review/` | Gate D | 审查 findings、复审报告 |
| `reviews/` | Gate D | 审查报告（备用路径） |
| `tmp/` | 全部 | 过程临时产物（截图、快照等），不入版本库 |

## 日期目录（按迭代批次）

| 目录 | 说明 |
|------|------|
| `2026-05-11/` | 上一迭代批次产物（架构、计划、需求、技能、任务） |
| `2026-05-12/` | 当前迭代批次产物（架构、实现、计划、需求、审查、任务、测试） |

> 日期目录内的子目录与阶段目录名一致，按迭代批次组织产物文件。

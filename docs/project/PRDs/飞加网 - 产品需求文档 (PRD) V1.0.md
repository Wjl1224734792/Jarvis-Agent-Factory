# 飞加网 - 产品需求文档（PRD）V1.0

## 1. 项目背景与目标

* **背景：** 低空经济快速发展，无人机、飞行汽车等受众扩大，但缺乏专业数据库与用户口碑结合的垂直社区。
* **目标：** 建立PC端优先的WEB应用（兼顾移动端），让飞友便捷地浏览飞行器参数、查看真实用户评分与点评、分享飞行体验，并参与社区互动。支持用户创建和参与榜单，通过关注机制增强社交属性，同时为厂商提供市场洞察。

---

## 2. 术语表

| 术语    | 定义                                        |
| ----- | ----------------------------------------- |
| 帖子    | 用户发布的图文/视频内容，包含短内容（类似动态）和长内容（类似文章），通过标签区分 |
| 点评    | 用户对机型的评分（1-5星）及可选文字，每个用户对同一机型只有一条点评记录     |
| 综合评分  | 基于所有用户对机型的评分加权平均后转换为10分制显示（如9.2分）         |
| 榜单    | 基于综合评分或其他维度生成的排行列表，支持用户自定义创建              |
| 关注    | 用户之间单向订阅关系，关注后可在首页“关注”Tab看到对方帖子           |
| 点赞    | 对内容表示赞同，按钮颜色变为主色调                         |
| 收藏    | 保存内容以便日后查看，按钮颜色变为橙色，存入默认收藏夹               |
| 认证账号  | 经平台官方认证的账号（品牌方、媒体、KOL等），拥有专属标识            |
| 私信    | 用户之间一对一的实时聊天                              |
| 分享    | 将内容分享到站内私信或站外（复制链接）                       |
| 飞行器类型 | 按飞行器形态划分的大类，如无人机、固定翼、直升机、飞行汽车、公务机等        |
| 动力类型  | 按动力来源划分的类别，如电动、燃油、混合动力等                   |
| 快速评分  | 榜单或机型列表页嵌入的星级评分组件，用户可快速评分（无文字）            |
| 心愿单   | 用户标记“想买”意向的机型列表，与收藏区分开，用于意向统计与推荐          |

## 3. 用户场景

* **小白用户：** 查机型参数、看真实用户评分和点评、读新手入门指南。
* **资深飞友：** 分享航拍作品、参与机型讨论、创建个性化榜单、关注同好。
* **行业观察者：** 看最新资讯、行业榜单、技术分析、私信交流。
* **厂商/品牌方：** 发布官方资讯、回复用户咨询、收集反馈（通过认证账号）。

---

## 4. 整体布局（PC端）

采用**顶部导航栏 + 左侧边栏 + 主内容区**的经典PC布局。

* **顶部导航栏：** Logo、全局搜索框、**发布按钮（主色调）**、消息图标、用户头像（登录/注册入口）。
* **左侧边栏：** 固定宽度240px，包含一级菜单：首页、飞友圈、飞行器库、榜单、个人中心；底部有设置和下载APP（可配置）。
* **主内容区：** 最大宽度1200px，最小宽度800px，左右内边距24px，上下内边距24px。卡片背景纯白，圆角12px。

---

## 5. 视觉规范

* **主色调：** 科技蓝 (#1E88E5) —— 按钮、链接、选中态、点赞选中状态。
* **辅助色：**

  * 评分橙 (#F57C00) —— 所有评分显示、收藏选中状态。
  * 危险红 (#F44336) —— 删除、错误提示、低分。
  * 认证金 (#FFB300) —— 认证标识。
  * 赞同灰 (#757575) —— 点赞/收藏未选中状态。
* **背景色：** 页面主背景 #F8F9FA，卡片背景 #FFFFFF。
* **字体：** 一级标题 #333333，二级正文 #666666，辅助信息 #999999；标题16px，正文14px，辅助12px。
* **圆角：** 卡片12px，按钮/输入框8px，头像50%。

## 6. 功能模块详解

### 6.1 登录/注册

* **入口：** 顶部导航栏“登录”按钮或需要登录的操作时弹出模态框。
* **方式：** 手机号 + 短信验证码，前置图形验证码防刷。
* **首次设置密码或修改敏感信息**：大小写+数字+特殊符号，8-20位，可跳过。
* **账号注销：** 二次验证 → 提交申请 → 7天冷静期 → 正式注销。内容匿名化，评分保留但用户ID匿名。

### 6.2 首页信息流

* **路由：** `/home`
* **顶部标签：** 推荐、关注、最新、资讯、测评、航拍、技术、指南（后台可配置）。
* **内容展示：** 瀑布流卡片（单列/三列自适应）。

  * **图片区裁切：** 强制16:9比例，`object-fit: cover`。
  * **多图提示：** 小圆点或缩略条。
* **右侧悬浮按钮：** 发布帖子（快捷入口）。
* **推荐策略：**

  * 初期：热度+时间衰减（点赞、评论、收藏数）；
  * 中期：协同过滤推荐，结合“看过机型”“关注的人”“点评过的机型”。

### 6.3 飞行器库

* **路由：** `/models`
* **多维筛选布局：** 顶部类型Tabs + 左侧品牌索引 + 动力类型筛选。
* **筛选逻辑明确化：**

  * **动力类型多选**采用 **OR逻辑**（同时选中多个动力类型的机型均显示）。
  * **“不限”按钮**重置筛选。
* **左侧品牌索引：** A-Z字母索引，点击滚动到对应品牌。
* **右侧机型网格：** 每行3-4个卡片，顶部显示筛选条件下机型总数。
* **机型详情页：**

  * 核心参数卡片、详细参数表（新增字段：飞行器类型、动力类型）。
  * **按钮组：** 想买（心愿单）、写点评、收藏、分享。
  * 热门评测推荐（横向滑动卡片）。
* **用户点评区：**

  * 星级+文字输入，点评唯一。
  * 综合评分10分制，基于所有用户点评加权平均，后台可异步计算。

### 6.4 帖子详情与评论

* 支持图文/视频，评论区支持无限嵌套。
* 点赞、回复、编辑、删除、举报。
* 点赞即时前端更新，后台异步一致性保证。
* 敏感词过滤，评论列表支持分页/无限加载。

### 6.5 榜单

* **榜单排序规则：**

  * 官方榜：动态生成按 **综合评分+点评人数**，可引入贝叶斯平滑。
  * 用户榜：创建者可调整顺序。
* **口碑标签：**

  * 人工标签优先 > 自动标签兜底。
  * 自动标签规则：≥9.0“神机”，8.0-8.9“真香”，7.0-7.9“中规中矩”，6.0-6.9“有遗憾”，≤5.9“差评”。
* **快速评分组件：**

  * 点击星级直接评分，创建/更新点评，无文字。
  * 点评即时刷新综合评分。
* **添加机型：** 用户可申请添加，后台审核。
* **创建者管理：** 审核申请、调整排名顺序、编辑榜单信息。

### 6.6 个人中心

* **功能矩阵升级：**

  * 我的帖子
  * 我的点评
  * 我的榜单
  * 我的收藏
  * 浏览历史 **区分“帖子”和“机型”**
  * 消息中心
  * 设置（账号安全、隐私设置、退出登录）
* **用户主页：** `/user/:id` 展示他人信息，关注/取关、私信、内容Tabs。

### 6.7 消息系统

* 私信、通知，实时聊天。
* 消息列表分Tab，支持敏感词过滤、拉黑用户。
* 异步推送，保证最终一致性。

### 6.8 发布功能

* 顶部“发布”按钮，下拉：发帖子/创榜单。
* 内容审核：敏感词过滤、图片审核，审核结果通过消息通知。

## 7. 管理员后台

* **仪表盘：** 关键数据概览。
* **用户管理：** 列表、搜索、禁用/封号、认证标识。
* **内容审核：** 多维筛选（内容类型、状态、关联机型、举报次数）、批量操作。
* **机型管理：** 增删改查，参数模板关联飞行器类型。
* **分类管理：**

  * 飞行器类型管理：增删改查，支持排序、启用/禁用。
  * 动力类型管理：增删改查，支持排序。
  * 品牌管理：可关联飞行器类型。
  * 参数模板管理：为每类飞行器创建可选参数模板。
  * 文章/榜单分类管理、首页标签统一管理。
* **举报管理**：处理用户举报，下架内容并通知。
* **系统通知**：发布公告。

## 8. 非功能性需求

* **性能：** 首屏加载 < 2s，图片懒加载，虚拟滚动优化。
* **兼容性：** Chrome、Firefox、Safari、Edge 最新版。
* **安全：** 密码加密存储，接口防篡改，防XSS/CSRF，敏感操作二次验证。
* **内容安全：** UGC内容需敏感词和图片审核。
* **数据一致性与并发：** 点赞、评分、评论使用Redis或消息队列异步计数，综合评分定时更新。
* **可观测性：** 核心接口监控（可用性、延迟、错误率），日志系统。

---

## 9. 移动端适配要点

* **布局：** 底部固定导航栏，顶部简化栏。
* **飞行器库页面适配：**

  * 顶部类型Tabs、动力类型标签组：横向滚动。
  * 品牌索引：悬浮字母条或下拉收起。
  * 机型列表：双列网格。
* **交互优化：** 下拉刷新、上拉加载，右滑返回，模态框改底部弹窗。
* **断点：** 屏幕宽度 <768px。
* **功能一致性**：保留所有核心功能。
* **灰度策略：** 新UI上线前，先进行小范围A/B测试，收集用户操作数据和反馈，调整筛选交互与排序逻辑。

---

## 10. 风险与实施建议

1. **数据一致性与并发：** 高并发点赞/评分时，采用消息队列异步处理，保证最终一致性。
2. **飞行器类型动态扩展：** 参数模板关联类型，后台可选择或创建专属模板。
3. **排序平滑策略：** 榜单可使用贝叶斯平均避免评分偏差。
4. **移动端交互验证：** A/B灰度测试确保用户理解筛选逻辑。

## NPM Domain — CLI npm 运行方式

### TASK-NPM-001: CLI shebang 移除 + npm run 脚本

- **task_name**: CLI shebang 移除 + npm run 开发脚本
- **requirement_ids**: [REQ-NPM-001, REQ-NPM-004]
- **type**: 直接开发
- **priority**: P0（阻断）
- **estimated_lines**: ~30
- **test_strategy**: manual_only（手动验证 `npm run jarvis -- help` 和 `npx jarvis help`）
- **dependencies**: []
- **parallel_group**: A
- **risk**: 低
- **涉及文件**:
  - `bin/jarvis.js` — 移除 `#!/usr/bin/env node` shebang，添加"仅供 npm bin 包装器调用"注释

- **实现 Agent 类型建议**: backend-dev-expert（简单文件修改）
- **验收标准**:
  1. `bin/jarvis.js` 首行不再包含 `#!/usr/bin/env node`
  2. `package.json` scripts 包含 `jarvis` 和 `jarvis:dev` 条目
  3. `npm run jarvis -- help` 正常执行不报错
  4. `npm run jarvis:dev -- help` 正常执行不报错
  5. 现有 `npm install -g` 安装后 `jarvis` 命令仍可正常使用（npm 自动生成 .cmd 不依赖 shebang）

- **requirement_ids**: [REQ-DASH-003]
- **type**: TDD
- **priority**: P1
- **estimated_lines**: ~250
- **test_strategy**: tdd（归档/恢复/删除的单元测试 + test_after 手动验证 UI）

- **shared_files**: db.ts, routes.ts, server.ts, pipeline.html（均与 DASH-001/002/004 共享）
- **parallel_group**: D（与 DASH-004、DASH-005 串行）
- **risk**: 高（涉及 5 个文件、新页面创建、数据删除操作需确认防误删）
- **涉及文件**:
  - `src/engine/db.ts` — `initSchema()` 添加 `archived INTEGER DEFAULT 0` 列迁移；新增 `archiveRun(db, runId)`、`unarchiveRun(db, runId)`、`getArchivedRuns(db)`、`getArchivedRunsBySession(db, sessionId)`、`deleteRun(db, runId)` 函数
  - `src/engine/server.ts` — `startWeb()` 中注册 `GET /archive` 路由返回 `archive.html`
  - `src/web/routes.ts` — 注册 `POST /api/pipeline-runs/:id/archive`、`POST /api/pipeline-runs/:id/unarchive`、`GET /api/pipeline-runs/archived`、`DELETE /api/pipeline-runs/:id` 路由
  - `src/web/views/pipeline.html` — `renderSessions()` 卡片添加三点菜单（⋮），含「归档」「删除」选项；归档后卡片从主列表消失
  - `src/web/views/archive.html` — **新文件**：归档页面（按 session 分组的 run 列表，显示 task_name、指令标签、归档时间；支持「恢复」「删除」操作按钮；支持按 task_name 搜索）
- **实现 Agent 类型建议**: frontend-dev-expert（前端为主）+ backend-dev-expert（API 和数据层可由同一 Agent 完成，或拆分为前后端两个子任务在实现阶段处理）

  1. `pipeline_runs` 表新增 `archived INTEGER DEFAULT 0` 列
  2. `POST /api/pipeline-runs/:id/archive` → archived=1 且活跃会话列表不再显示该 run
  3. `POST /api/pipeline-runs/:id/unarchive` → archived=0 恢复显示
  4. `GET /api/pipeline-runs/archived` 返回所有已归档 run
  5. `DELETE /api/pipeline-runs/:id` 硬删除 run（前端弹出确认对话框，`confirm('确认永久删除该运行记录？')` 防误操作）

  7. 归档页面支持 task_name 搜索过滤
  8. 活跃会话列表中被归档的 run 不再出现
  9. 单元测试覆盖：archive → verify archived=1；unarchive → verify archived=0；delete → verify record gone

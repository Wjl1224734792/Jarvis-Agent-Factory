# 移动端 / 微信小程序前端开发文档

> 适用对象：移动端 App、微信小程序前端开发  
> 依赖文档：
> - [OpenAPI 前端对接指南](/E:/CodeStore/feijia/docs/openapi-frontend-integration-guide.md)
> - `GET /docs`
> - `GET /openapi.json`

## 1. 文档目标

本文档用于指导移动端与微信小程序前端落地开发，覆盖：

- 布局规范
- 视觉规范
- 页面结构
- 页面逻辑
- 功能逻辑
- 接口接入顺序
- 状态管理建议

目标是让你可以只依赖本文档和 OpenAPI 文档开始开发，而不需要反复回看 Web 页面代码。

## 2. 客户端范围

本期客户端范围包含：

- 登录 / 注册补全
- 首页
- 飞友圈
- 飞行器库
- 榜单
- 发布中心
- 用户主页
- 设置
- 通知

不包含：

- 管理端
- 支付
- 聊天
- 微信原生授权登录
- 后台审核能力

## 3. 技术建议

### 3.1 微信小程序

- 建议使用 `Taro`
- 语言：TypeScript
- 状态管理：Zustand 风格或轻量 store
- 网络层：统一 request 封装

### 3.2 移动端 App

- 建议使用 `Flutter`
- 如果只是先做一版统一前端，也可以和小程序一起用 `Taro + React Native` 思路复用信息结构，但不是本仓库推荐主线

## 4. 视觉与布局规范

### 4.1 总体原则

- 使用移动端布局，不复刻 Web 右侧边栏。
- 避免大面积边框线。
- 用留白、分组背景、阴影、圆角和间距做层级。
- 优先使用块状分区，而不是 `border` 分隔。

### 4.2 页面基础结构

统一采用：

1. 顶部导航区
2. 页面内容滚动区
3. 底部主导航

其中：

- 顶部导航固定
- 内容区独立滚动
- 发布页可以隐藏底部导航，改为独立沉浸式页面

### 4.3 样式原则

- 卡片使用圆角块面
- 少边框，弱分隔
- 重要操作使用主色按钮
- 次要操作使用文字按钮或浅底按钮
- 评论、互动、标签、统计统一轻量化

### 4.4 不建议的样式

- 不要用 Web 那种左右双栏布局
- 不要依赖细边框堆页面层次
- 不要在移动端保留过多并列按钮
- 不要把复杂编辑器工具栏完整照搬 Web

## 5. 信息架构

建议底部 5 个一级入口：

1. 首页
2. 飞友圈
3. 发布
4. 榜单
5. 我的

飞行器库不放底部一级入口，而放在首页二级入口或搜索入口中。

## 7. 页面布局建议

### 7.1 首页

布局：

- 顶部搜索
- 推荐 / 最新 / 关注切换
- 内容流
- 热门榜单横向卡片
- 热门机型纵向紧凑卡片

说明：

- 热门机型卡片显示：
  - 缩略图
  - 机型名
  - 品牌名
  - 点赞数
  - 评论数
- 不显示 Web 里的测评数旧文案

### 7.2 飞友圈

布局：

- 顶部 Tab
- 瀑布流卡片
- 点击进入详情

卡片信息：

- 封面图 / 视频角标
- 标题
- 作者
- 点赞数

详情页：

- 媒体区
- 标题 / 正文
- 评论区
- 评论排序：最新 / 热门

### 7.3 飞行器列表

布局：

- 搜索框
- 分类筛选抽屉
- 卡片列表

卡片信息：

- 图片
- 品牌
- 机型名
- 价格
- 一句话摘要

不要在移动端保留 Web 那种右侧筛选栏。

### 7.4 飞行器详情

布局：

- 图片轮播
- 核心标签
- 标题 / 价格 / 简介
- 参数区
- 互动区
- 评论区

评论区必须带：

- 最新
- 热门

### 7.5 榜单列表

布局：

- 榜单分类切换
- 榜单卡片流

卡片信息：

- 封面
- 标题
- 描述
- 条目数量 / 评论数

### 7.6 榜单详情

布局：

- 榜单封面
- 标题 / 描述
- 条目列表
- 评论区

### 7.7 评分对象详情

布局：

- 对象封面
- 标题 / 品牌 / 摘要
- 评分总览
- 分项评分
- 评论区

评论区必须带：

- 最新
- 热门

### 7.8 设置页

布局建议改成纵向单列卡片：

- 头像
- 昵称
- 个人简介
- 可见范围
- 绑定手机
- 通知设置

交互原则：

- 每项单独编辑
- 每项单独保存
- 手机号换绑使用弹窗

### 7.9 发布页

统一原则：

- 顶部只保留必要操作
- 主体用单列表单
- 预览可做成底部抽屉或独立页
- 不保留 Web 右侧预览栏

## 8. 发布功能逻辑

### 8.1 发布文章

字段：

- 标题
- 分类
- 封面
- 摘要
- 正文

规则：

- 摘要限制 100 字
- 摘要不可手动拖拽高度
- 正文富文本工具栏一排排列
- 封面改成点击大框上传

### 8.2 发布动态

字段：

- 标题可选
- 内容必填
- 图片或视频

规则：

- 图片张数不再限制 6 张
- 图片与视频不能混发
- 评论区详情默认按最新排序

### 8.3 发布飞行器

字段：

- 分类
- 机型名称
- 品牌
- 动力
- 状态
- 封面
- 价格
- 一句话摘要
- 长描述
- 参数

规则：

- 只能从已有品牌中选择
- 不再提供品牌申请跳转按钮
- 一句话摘要限制 50 字
- 长描述限制 300 字
- 两个文本框都禁用拖拽高度
- 封面通过大框点击上传
- 状态字段统一：
  - 概念
  - 研发
  - 测试
  - 未发布
  - 已发布
  - 未上市
  - 已上市

### 8.4 申请品牌

字段：

- 品牌名称
- slug
- 描述
- Logo 可选

规则：

- Logo 点击左侧图片区上传
- slug 由用户手动输入英文名
- 描述限制 500 字
- 描述禁用拖拽高度
- 除 Logo 外全部必填

### 8.5 创建榜单

字段：

- 标题
- 简介
- 封面
- 排行对象

规则：

- 主封面初始不显示默认图
- 点击大框上传封面
- “自定义条目”统一文案为“添加排行对象”
- 飞行器库候选列表要放在卡片内部滚动区

## 9. 评论区统一逻辑

所有评论区统一遵循：

- 默认排序为 `最新`
- 支持切换到 `热门`
- 支持：
  - 发表评论
  - 回复
  - 点赞
  - 举报
  - 作者编辑
  - 作者删除

适用页面：

- 文章详情
- 飞友圈动态详情
- 飞行器详情
- 评分对象详情

## 10. 交互逻辑

### 10.1 登录态拦截

以下操作未登录时统一弹登录引导：

- 点赞
- 收藏
- 分享
- 评论
- 回复
- 举报
- 发布
- 关注

### 10.2 错误提示

优先级：

1. 字段级错误
2. 页面级错误
3. 底部轻提示

必须做字段级提示的场景：

- 用户名已被占用
- 图形验证码错误
- 短信验证码错误
- slug 非法

### 10.3 上传

移动端前端必须统一封装上传层，不要在页面里直接散写：

- 初始化上传
- 直传
- 完成回写

## 11. 状态管理建议

建议拆 4 层：

- `auth`
  登录态、用户资料、token
- `feed`
  首页 / 飞友圈列表
- `publish`
  各发布页草稿状态
- `comment`
  评论排序、回复态、草稿态

## 12. 页面开发顺序

推荐顺序：

1. 登录 / 注册补全
2. 首页
3. 飞友圈
4. 飞行器列表 / 详情
5. 榜单 / 评分对象
6. 我的 / 设置 / 通知
7. 发布中心

## 13. 交付建议

为了方便和当前接口对接，建议你先产出这些前端基础模块：

- 认证模块
- 上传模块
- 评论通用模块
- Feed 通用卡片
- 发布通用表单组件
- 错误码映射模块

如果这几个基础件先搭好，后面的页面开发速度会快很多。

## TDD 验证范围

由于这是 CSS 修复，TDD 策略与传统单元测试不同：

1. **视觉回归基准**：录制修复前滚动行为（不可滚动），验证修复后可滚动
2. **CSS 属性断言**：计算样式验证 `overflow: auto` / `flex: 1` 正确应用
3. **跨 Tab 滚动重置**：切换分类/来源 Tab 后滚动位置归零
4. **布局隔离验证**：确认不影响其他页面（Layout 级 `overflow: hidden` 必须保留，仅内容区独立滚动）

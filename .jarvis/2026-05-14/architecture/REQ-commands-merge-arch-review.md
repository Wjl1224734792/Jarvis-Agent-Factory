# REQ-Commands-Merge 架构评审

**日期**: 2026-05-15
**结论**: 无架构变更，现有架构完全支持

---

## 评审结论

本需求为现有 Web 面板指令页面的功能增强，**不涉及任何架构变更**：

- **前端**: 在现有 React + React Router 架构内，修改单一页面组件 `Commands.tsx`，增加了 Tab 切换（Ant Design `Tabs` 组件已在项目中使用）
- **后端**: 在现有 Hono 路由中扩展 `/api/commands` 端点，仅增加文件读取源，不改变数据流
- **数据**: 纯读操作，无数据库 Schema 变更，无新增存储
- **API 契约**: 返回结构从 `{ commands: [], total: number }` 改为 `{ project: {...}, global: {...} }`，是后向不兼容变更，但该 API 仅被 Web 面板自身消费

## 风险

| 风险 | 级别 | 缓解 |
|------|------|------|
| API 返回格式变更 | 低 | 仅内部消费，前后端同步发版 |
| 全局目录不存在 | 低 | 返回空数组兜底 |
| 文件读取权限 | 低 | 同一用户目录，无跨用户访问 |

## 建议

- 前后端变更在同一个 commit 中发布，避免 API 契约不一致
- 无需独立的前端/后端架构评审子文档

# 评论区发布时间 + 操作条交互统一

## 问题根因

- [model-comments-section.tsx](apps/web/src/routes/model-comments-section.tsx)、[post-comment-thread.tsx](apps/web/src/features/posts/post-comment-thread.tsx)、[rating-target-detail-page.tsx](apps/web/src/routes/rating-target-detail-page.tsx) 的元信息行均使用 **`formatTime(...updatedAt)`**；后端在互动后往往会刷新 `updatedAt`，界面时间随之变化。
- 帖子嵌套回复的 [ReplyView](apps/web/src/features/posts/post-comment-thread.tsx) 只保留了 `updatedAt`，未带 `createdAt` / `viewer`，嵌套行既无法固定「首发时间」，也无法给点赞做选中态。

## 1. 时间与展示逻辑

**新增** [apps/web/src/lib/comment-relative-time.ts](apps/web/src/lib/comment-relative-time.ts)（或 `lib/relative-time.ts`）：

- `formatCommentPublishedTime(iso: string, now?: Date): string`：纯函数，**只读 `createdAt`**，相对区间与绝对区间规则如下：
  - **&lt; 1 分钟** → `刚刚`
  - **&lt; 60 分钟** → `N 分钟前`
  - **&lt; 24 小时** → `N 小时前`
  - **&lt; 7 天** → `N 天前`
  - **否则（久远）** → `zh-CN` **仅年月日**，**不显示时分**，例如 `2026/4/7`（可用 `toLocaleDateString('zh-CN', { year: 'numeric', month: 'numeric', day: 'numeric' })` 等与项目一致的日期格式，避免与原先「全量 `toLocaleString` 含时分」混用）。
- `useCommentPublishedTimeLabel(createdAt: string)`：`useState` + `useEffect`，**每 60s** 用 `Date.now()` 重算相对文案；挂载时立即算一次。

**替换展示字段**（三文件元信息行、帖子嵌套回复行）：

- 全部改为 **`comment.createdAt` / `reply.createdAt`**，删除对 `updatedAt` 的展示依赖。
- 帖子：`ReplyView` 增加 `createdAt`、`hasLiked`（来自 `postCommentReplySchema` 的 `createdAt` / `viewer.hasLiked`）；`flattenReplies` 映射补齐。
- `InteractionRow` 增加 **`hasLiked?: boolean`**（根评从 `comment.viewer.hasLiked`，嵌套从 `reply.hasLiked`），供点赞图标选中态使用。

## 2. 统一操作条控件（图标 vs 文字）

**新增** [apps/web/src/components/comment-thread-controls.tsx](apps/web/src/components/comment-thread-controls.tsx)（命名可微调），封装与 [Button](apps/web/src/components/ui/button.tsx) + `group` 的组合，避免三套重复 class：

| 控件 | 行为 |
|------|------|
| `CommentLikeIconButton` | `border-0 bg-transparent hover:!bg-transparent active:translate-y-0`；心形 `transition-transform group-active:scale-[0.92]`；`hasLiked` 时 `fill` + 语义色（如 rose） |
| `CommentIconOnlyButton` | 编辑/删除共用：仅图标缩放 + hover 图标色略加深，**无背景块** |
| `CommentTextAction` | 用于「回复」「举报」：`text-muted-foreground transition-colors`；回复 `hover:text-primary`；举报 `hover:text-orange-600`（dark 适配） |

举报若需「已举报」弱提示，可对 `hasReported` 加 `text-orange-600/90`（与评分对象页举报图标策略一致，可选）。

**接入点**：

1. [post-comment-thread.tsx](apps/web/src/features/posts/post-comment-thread.tsx)：`InteractionRow` 改为使用上述组件；传入 `hasLiked`、`isEditing`（编辑中可对笔图标做轻微选中/加粗）；`reportTrigger` 内 `Button` 改为 `CommentTextAction` 或包一层相同 class。根评 `InteractionRow` 传入 `props.comment.viewer.hasLiked`；嵌套传入 `reply.hasLiked`。
2. [model-comments-section.tsx](apps/web/src/routes/model-comments-section.tsx)：`CommentActions` 内点赞/编辑/删除/回复/举报触发器替换为同一套组件；编辑需从 `ModelCommentCard` 传入 `isEditing`。
3. [rating-target-detail-page.tsx](apps/web/src/routes/rating-target-detail-page.tsx)：内联 `CommentActions` 同上；逻辑与 API 不变。

删除按钮按与编辑相同的「仅图标」规则处理，保证整行视觉一致。

## 3. 验证

- `bun run typecheck`（`apps/web`）。
- 手动：对一条评论点赞后，**时间文案不变**；相对时间在同一页停留约 1 分钟后应能从「N 分钟前」合理递增（基于 `createdAt`）。
- 点赞/编辑/删除：无 ghost 大背景块，仅有图标缩放与填充/颜色；回复/举报：悬停文字变色。
- 超过 7 天的评论：元信息仅显示 **年月日**，无时分。

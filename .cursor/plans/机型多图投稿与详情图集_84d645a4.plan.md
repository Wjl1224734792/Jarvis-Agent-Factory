# 发布飞行器多图 + 机型详情可滚动图集

## 背景结论

- 投稿侧：[`publish-aircraft-page.tsx`](D:\CodeStore\feijia\apps\web\src\routes\publish-aircraft-page.tsx) 已把 `galleryImageFileIds: []` 写死，且 [`handleImageUpload`](D:\CodeStore\feijia\apps\web\src\routes\publish-aircraft-page.tsx) 只保留一张图；后端 [`aircraft-submissions`](D:\CodeStore\feijia\apps\server\src\modules\aircraft-submissions\aircraft-submissions.service.ts) 与 Zod [`galleryImageFileIds` max(6)](D:\CodeStore\feijia\packages\schemas\src\aircraft-submissions.ts) 已支持多图。
- 机型侧：[`aircraft_models`](D:\CodeStore\feijia\packages\db\src\schema.ts) **没有** `coverImageFileId` / `galleryImageFileIds`；审核通过时 [`createModel`](D:\CodeStore\feijia\apps\server\src\modules\aircraft-submissions\aircraft-submissions.service.ts) 未拷贝投稿图片。
- 详情页：[`model-detail-page.tsx`](D:\CodeStore\feijia\apps\web\src\routes\model-detail-page.tsx) 使用 [`getModelGallery(..., 4)`](D:\CodeStore\feijia\apps\web\src\lib\aviation-media.ts) 占位，缩略区为 `grid grid-cols-4`，无法展示多于 4 张且无真实数据。

## 数据层

1. **Drizzle schema**（[`packages/db/src/schema.ts`](D:\CodeStore\feijia\packages\db\src\schema.ts)）为 `aircraft_models` 增加：
   - `coverImageFileId: text(...)` 可空  
   - `galleryImageFileIds: text(...).default("[]")`（与 `aircraft_submissions` 一致，JSON 字符串数组）

2. **生成并应用迁移**：在 `packages/db` 下用项目既有 Drizzle 流程生成新 migration（如 `0007_*.sql`），并更新 `drizzle/meta` journal；部署/本地执行 migrate。

3. **种子数据**（可选）：[`seed.ts`](D:\CodeStore\feijia\packages\db\src\seed.ts) 可为个别机型写入与现有关联的 file id，便于联调；无则详情页继续走占位 fallback。

## 后端：机型读写与审核拷贝

4. **Repo**（[`aircraft-models.repo.ts`](D:\CodeStore\feijia\apps\server\src\modules\aircraft-models\aircraft-models.repo.ts)）  
   - `findBySlug` / `findById` 的 `select` 增加上述两列。  
   - `create` / `update` 的 `input` 与 `values`/`set` 支持写入（可空 / JSON 字符串）。

5. **Service**（[`aircraft-models.service.ts`](D:\CodeStore\feijia\apps\server\src\modules\aircraft-models\aircraft-models.service.ts)）  
   - `createModel` / `updateModel` 方法签名增加 `coverImageFileId`、`galleryImageFileIds: string[]`（内部 `JSON.stringify`）。  
   - `getModelDetail`：在返回给路由前，用现有 [`resolveUploadedFileUrl`](D:\CodeStore\feijia\apps\server\src\modules\uploads\uploads.helpers.ts) 解析封面 + 图集（可抽小函数 `parseGalleryJson`，与投稿侧 `parseGallery` 行为一致），合并进响应对象。

6. **审核流**（[`aircraft-submissions.service.ts`](D:\CodeStore\feijia\apps\server\src\modules\aircraft-submissions\aircraft-submissions.service.ts)）  
   - 首次 `createModel` 时传入 `current.coverImageFileId` 与 `parseGallery(current.galleryImageFileIds)`。  
   - `syncApprovedModelVisibility` 与所有 `updateModel` 调用处同步传入上述字段（从当前 submission 行读取），避免更新机型时把图清空。

7. **Admin 机型**（若存在创建/更新接口）：[`apps/admin` models API](D:\CodeStore\feijia\apps\admin\src\features\models\models-page.tsx) 与对应 server admin route — 为 `create`/`update` 增加**可选**封面/图集字段（可默认 `null`/`[]`），避免 Zod/API 校验失败。若 admin 暂不编辑图片，服务端默认值即可。

## 契约与类型

8. **Zod**（[`packages/schemas/src/models.ts`](D:\CodeStore\feijia\packages\schemas\src\models.ts)）在 `modelDetailSchema` 上增加：  
   - `coverImageUrl: z.string().nullable()`（或 `.optional()` 与默认 `null` 一致）  
   - `galleryImageUrls: z.array(z.string().min(1)).default([])`  

9. **测试**：更新 [`packages/schemas/tests/models.test.ts`](D:\CodeStore\feijia\packages\schemas\tests\models.test.ts) 中 `modelDetailResponseSchema` 样例，带上新字段。

10. **Web `api-client`**：若 [`WebModelDetailResponse`](D:\CodeStore\feijia\apps\web\src\lib\api-client.ts) 手写类型，与 schema 对齐。

## 前端：发布页

11. **状态**：拆成 `coverImage: UploadedImage | null` 与 `galleryImages: UploadedImage[]`（最多 6 个，与投稿 schema 一致）；与视频互斥逻辑保持（选视频则清空图集）。

12. **上传**：继续调用 `uploadAircraftCoverImage` 逐张上传；支持 `input multiple` 批量选择；首张写入封面（若尚无封面），其余追加到图集直至上限；提供删除单张、可选「设为封面」（把该张与当前封面交换）以提升可用性。

13. **提交**：`coverImageFileId: coverImage?.id ?? null`，`galleryImageFileIds: galleryImages.map(i => i.id)`（过滤与封面重复 id）；编辑模式从 `getAircraftSubmission` 回填 `coverImageUrl`/`galleryImageUrls`（及 id，若接口已返回 `galleryImageFileIds`）。

14. **文案**：更新「仅一张图」类提示；侧栏预览可显示小缩略图条（可选）。

## 前端：机型详情

15. **图集数组**：用 `item.coverImageUrl` 与 `item.galleryImageUrls` 合并为有序列表（封面优先，图集去重）；若结果为空，再 `fallback` 到 `getModelGallery(item.slug, item.powerType, 4)`（保持旧数据有图可看）。

16. **主图**：`src={gallery[activeGalleryIndex] ?? ...}`；`slug` 变化时 `useEffect` 将 `activeGalleryIndex` 置 0。

17. **缩略条**：将 `grid grid-cols-4` 改为横向 **`flex overflow-x-auto gap-2`**（可加 `pb-1`、`scrollbar-thin` 或 `scroll-smooth`）；每个 thumb **`shrink-0`** 固定宽度（如 `w-20` / `h-16`），选中态边框逻辑不变；支持任意张数滚动。

## 验证建议

- 发布页：选 1 封面 + 多张图，提交后 DB submission 中 `gallery_image_file_ids` 非空。  
- 审核通过后：`aircraft_models` 行含相同 file id；`GET /models/:slug` 返回非空 `galleryImageUrls`。  
- 详情页：多图时横滑缩略图，点击切换主图；无图机型仍为占位四图。  
- `bun run --cwd packages/schemas test`（如有）、`apps/web` / `apps/server` 相关 typecheck。

# 他人主页布局对齐个人中心

## 现状差异

| 区域 | 个人中心 [`profile-page.tsx`](apps/web/src/features/auth/profile-page.tsx) | 他人主页 [`user-profile-page.tsx`](apps/web/src/routes/user-profile-page.tsx) |
|------|----------------|----------------|
| 头图 | `h-40 md:h-48`，渐变 `from-slate-950/80 via-slate-950/30`，**底部绝对定位**：`UserAvatar`（大）+ **白色大标题** + 可见性 Badge | 头图 + 较弱渐变，**无**头像/昵称叠在图上；头像 `-mt-10` 压在白底与头图交界 |
| 面板 | `SitePanel` `!border-0` + `variant="floating"` | 有 `variant="floating"`，**缺少** `!border-0` |
| 主体 | `lg:grid-cols-[minmax(0,1fr)_24rem]`：**左**简介（`md:grid-cols-[9rem_minmax(0,1fr)]` 左栏占位 + bio），**右**三列 `MetricStrip` | 左为头像+徽章+昵称+说明，右为指标（结构不同） |
| 下方 | **`Tabs` `variant="line"`**：「内容」「收藏」+ 分类/状态筛选 + `VirtualFeed` `className="!border-0"` | **无 Tabs**，直接 `VirtualFeed`（带边框样式与自主页不一致） |

公开资料接口 [`userProfileSchema`](packages/schemas/src/social.ts) 的 `user` 为 `userSummary`，**无 bio 字段**；对齐布局时左栏可继续用现有说明文案（或后续再接 `bio` 扩展），**不强制改协议**。

## 实现要点

### 1. 重构 `UserProfilePage` 头图与 `SitePanelBody`

- 与 `ProfilePage` 同步：
  - `SitePanel`：`className="overflow-hidden !border-0" variant="floating"`。
  - 头图容器内：封面图 + 与自主页一致的渐变层。
  - **底部** `absolute inset-x-0 bottom-0 p-5 md:p-6`：`flex items-end gap-4`，使用 [`UserAvatar`](apps/web/src/components/ui/user-avatar.tsx)（`!h-28 !w-28 md:!h-32 md:!w-32`）+ 昵称（`text-[2rem] md:text-[2.5rem]`、`text-white`、`drop-shadow`）+ 可见性相关 `Badge`（样式参考自主页 `border-white/24 bg-white/12 text-white`）。
  - 他人主页无「编辑封面」按钮，**不**增加上传控件。
  - 封面图：在 **`canViewProfile`** 为 true 时继续使用 `getProfileBanner`；若与自主页一致需真实封面，属后续增强，本次仍以现有数据源为准。

- **`SitePanelBody`**：改为与 `ProfilePage` 相同栅格——**左**侧仅放说明段落（现有 `canViewProfile` / `canViewContent` 文案即可），**右**侧 `MetricStrip` 三列；其下 **`border-t` + 操作区**（关注 / 返回我的主页），类名与自主页按钮行对齐。

### 2. 内容区：Tabs + 列表

- 引入 `Tabs` / `TabsList` / `TabsTrigger` / `TabsContent`（与 `profile-page` 相同 `variant="line"`）。
- **「内容」Tab**：内层放 `VirtualFeed`，并加 `className="!border-0"`，与自主页一致。
- **「收藏」Tab**：他人场景下**不能**展示对方收藏列表。推荐二选一（实现时选一种即可）：
  - **A**：第二个 `TabsTrigger` `disabled`，`title` 说明「无法查看他人收藏」；或
  - **B**：保留可点，`TabsContent` 内用 `Alert`/短文案说明不公开。

（若希望与自主页一样有分类 pill：**仅「内容」Tab** 内可增加与 `profile-page`「活动」类似的分类按钮，对 `contentItems` 做客户端过滤；需复用 [`filterProfileItems`](apps/web/src/features/auth/profile-content-filters.ts) 与 `ProfileContentCategory`，常量可从 `profile-page` 抽一层到 `profile-content-filters` 或局部复制常量数组。**生命周期筛选（草稿/审核中…）对他人主页通常无意义**，建议**不**加，除非产品明确要求。）

### 3. 骨架屏 [`UserProfilePageRouteSkeleton`](apps/web/src/components/page-skeletons.tsx)

- 对齐 `ProfilePageSkeleton`：**头图 + 底部圆形头像与标题条占位**；`SitePanelBody` 内 **左文案 + 右三列指标** + **底部按钮条占位**；下方 **Tabs 横条占位 + 列表区**（可用 `Skeleton` 条或简化的 `VirtualFeed` 占位高度），避免改完布局后 Suspense/首屏骨架仍像旧版。

### 4. 可选小整理

- `MetricStrip` 在 `user-profile-page` 与 `profile-page` 重复；若改动面允许，可抽到 `@/components` 或 `site-shell` 旁的小模块，**非必须**。

## 验证

- `bun run --cwd apps/web typecheck`
- 手动打开 `/users/{uuid}` 与 `/profile`（登录）对比头图、栅格、Tabs、列表边框是否一致。
